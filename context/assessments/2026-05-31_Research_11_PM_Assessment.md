## C. Cross-cutting analysis

1. **The explicit-discriminator consensus is unanimous and maps cleanly.** Every system surveyed carries the sum-type tag as explicit data, never reflection: Akka's string *manifest* beside the bytes, Avro's single-key JSON union object, Protobuf's `oneof` field-number, EventStoreDB's "don't couple storage to runtime types." This is the same philosophy HomeSynapse already uses one level up (`events.event_type` + `EventTypeRegistry`, never `@JsonTypeInfo`). REC-100's `{"t":<AttributeType>,"v":‚Ä¶}` is the device-value-layer twin of that decision. **Coherence with the codebase is exact** ‚Äî and stronger than the research realized (see ¬ßF corroboration #1).

2. **Determinism is the real risk surface, and it is a *float* risk, not a *map* risk.** The brief had already cleared map-ordering (no `Map` in any `AttributeValue` variant; `ArrayValue` is an ordered `List`). The research confirms that and correctly redirects the entire nondeterminism concern onto (a) float text rendering across JVM versions and (b) envelope field ordering (controlled trivially by writing fields in fixed order in a hand-rolled codec). This matches the source reality precisely: `FloatValue(double)` has **no constructor guard** and can carry `NaN`/`¬±Inf`; `QuantityValue` rejects non-finite at construction. So only the `FloatValue` arm needs the sentinel path ‚Äî the research states this asymmetry correctly.

3. **The upcast-vs-rebuild distinction is the conceptual backbone of G2.** The literature draws a sharp line ‚Äî Axon "payload conversion at handling time" (read-time upcast of a stored event) vs `TrackingEventProcessor` reset (rebuild the read model from the log). This maps 1:1 onto HomeSynapse's two surfaces: **Path A** = projection rebuild (re-derive typed state from the immutable `state_reported` log; rides AMD-50/AMD-51 unchanged; authoritative); **Path B** = read-time upcast of a historical String `state_changed` (today ‚Üí lossy `DegradedEvent`). The research's clean separation is exactly what AMD-52 needs to specify the two surfaces independently.

4. **Backward-compatibility framing matters for replay.** Kafka Schema Registry's BACKWARD mode is the right contract analogue: "rewinding consumers to the beginning of the topic" *is* replay, and BACKWARD guarantees the new reader can read old writes. HomeSynapse's `schema_version=1 ‚Üí typed` migration is therefore a backward-compatible read concern, which the existing `FAIL_ON_UNKNOWN_PROPERTIES`-disabled posture (INV-ES-07) already supports.

5. **Inline-BLOB is over-determined for Pi-class.** SQLite's own "35% faster than the filesystem" benchmark + the universal "opaque blob + indexed metadata columns" event-sourcing consensus + the explicit Marten/Postgres-JSONB contrast (rich server-side query HomeSynapse neither has nor wants on a Pi) all point one way. The constrained-hardware lens *strengthens* the decided choice rather than threatening it. The only live concern is GC pressure under the 256 MiB heap, which the compact `NON_NULL` envelope directly addresses.

6. **The anti-pattern set is real and sourced.** Never-mutate-the-log (Kurrent, Azure), no-lossy-upcast (Azure "bad events persist"), no-serializer-nondeterminism (the JWT MAC break, the glTF map-order break). REC-105's checklist operationalizes these into an author/coder gate. This is the kind of process artifact that prevents the M2.2/M2.4-class latent defect.

## E. Per-REC adjudication (REC-100..105)

### REC-100 ‚Äî Compact tagged-union envelope `{"t":<AttributeType>,"v":<value>}` (Fork 1, gate G1) ‚Äî **ACCEPT**
Sound and source-corroborated (Akka manifest + Avro JSON union). Honors every constraint: explicit `AttributeType` string discriminator (not reflective type info), one hand-rolled (de)serializer pair confined to `com.homesynapse.persistence`, exhaustive `switch` over the sealed 8-variant type (permitted ‚Äî D-01 is event-type-scoped; AMD-51-INV-01 already uses one), `ArrayValue` recurses the same envelope per element (ordered List preserved), `Include.NON_NULL` posture. **Source-verified corroboration the research missed:** `PersistenceJacksonModule` and `PersistenceObjectMapper` Javadocs *explicitly pre-declare* this exact expansion ‚Äî *"`AttributeValue` serde ‚Äî deferred to the state-store milestone (DECIDE-M2-03). No current event record uses the type."* So REC-100 is not a new design; it is the cash-out of a registered, source-anchored decision. **Closes G1's mechanism + envelope-shape sub-fork.** One authoring refinement: use `JsonSerializer`/`JsonDeserializer` (the repo precedent, see ¬ßF #2), not `StdSerializer`.

### REC-101 ‚Äî Pin a canonical float renderer; anchor hashes on the bit pattern (Fork 2, gate G1/G2) ‚Äî **ACCEPT (the determinism keystone)**
This is the highest-value REC and the PM concurs fully. The JDK 18‚Üí19 `Double.toString` change is a real, dated threat to byte-stability. **Decision the team must make explicit (the research's Caveat 2 is correct to flag it):** *is forensic identity defined over the decoded `double` value, or over the exact stored bytes?* PM lean ‚Äî **define identity over the decoded value and anchor any `chain_hash` contribution on `Double.doubleToLongBits` (bit pattern), not the rendered text.** This (a) makes replay determinism robust to a future Corretto/JDK upgrade across the Schubfach boundary, (b) reuses the exact AMD-51 ¬ß5 #5b `Double.doubleToLongBits` bit-identity discipline already in the codebase, and (c) still permits a pinned shortest-round-trip renderer for the human-readable stored `"v"` text without that text being load-bearing for the hash. **Caveat:** REC-101 option (b) "hash the bits not the text" *touches the reserved `chain_hash` derivation* ‚Äî the research correctly flags (¬ß5 #1) that this needs Nick's sign-off, not silent implementation. **This is a Nick decision item, surfaced in the fork memo.** Closes G1's float-rendering sub-fork and is a G2 precondition.

### REC-102 ‚Äî Sentinel strings for non-finite `FloatValue`; ban `ALLOW_NON_NUMERIC_NUMBERS` on the wire (Fork 2, gate G1) ‚Äî **ACCEPT**
Correct and necessary. Standard JSON cannot carry `NaN`/`¬±Inf`; bare tokens (Jackson `ALLOW_NON_NUMERIC_NUMBERS`, Python `json.dumps`) produce invalid JSON that breaks the forward-compat reader posture. Sentinel strings (`"NaN"`/`"+Inf"`/`"-Inf"`, optionally `"-0.0"`) inside `"v"` with a strict decoder is the cross-language consensus. **Source-verified:** the asymmetry the REC relies on is real ‚Äî `FloatValue` has no finite-guard (verified, no compact ctor), `QuantityValue` rejects non-finite at construction (verified, lines 105‚Äì108). So only the `FloatValue` arm needs the sentinel branch. **Two authoring sub-decisions the research correctly leaves open (¬ß5 #3/#4):** (i) NaN bit-pattern canonicalization ‚Äî a single `"NaN"` sentinel collapses signaling/quiet/payload NaNs (acceptable for smart-home, lossy in the strict bit sense ‚Äî confirm); (ii) `‚àí0.0` vs `+0.0` ‚Äî note the coherence point: **AMD-51 ¬ß2.3 maps `‚àí0.0`‚Üí`+0.0` in the comparator**, so if the serializer preserves `‚àí0.0` as a distinct sentinel, the stored form distinguishes a value the comparator treats as equal. Not a contradiction (storage fidelity ‚âÝ comparison semantics), but AMD-52 must state the chosen stance. Closes G1's IEEE-edge sub-fork.

### REC-104 ‚Äî Wire `AttributeValueUpcaster` into the Path-B decode path (version-branched lift); retire `DegradedEvent` for legacy rows (Fork 4, gate G2) ‚Äî **ACCEPT WITH NARROWING**
This is the one genuinely open scope call and it carries real blast radius. The research's recommendation ‚Äî branch on `events.schema_version`, lift a `schema_version=1` String `state_changed` to a typed value rather than emitting a lossy `DegradedEvent`, keeping `upcastLenient(...)` as the last-resort fallback ‚Äî is well-reasoned (Axon "payload conversion at handling time" + Microsoft/Marten read-time-upcast endorsement) and the `AttributeValueUpcaster` SPI signature it targets is **source-confirmed verbatim** (`canUpcast(String, int)` / `upcast(String, String, int)` / `upcastLenient(...)` default).

**PM narrowing (important, and a ¬ß7 mechanism refinement):** the version branch must live at the **`EventPayloadCodec.decode(eventType, schemaVersion, payload)` level** ‚Äî which *has* the `schema_version` (source-confirmed: it is a method parameter threaded from the column) ‚Äî **not inside a nested Jackson `AttributeValueDeserializer`**, which sees only the JSON token stream and has no access to external row metadata unless `schema_version` is threaded in via `ObjectReader` context attributes. The research's ¬ß7 hedges this correctly ("the `AttributeValueDeserializer` *or the enclosing `StateChangedEvent` decode path*"), but AMD-52 must pick the decode-path locus explicitly. **Further narrowing:** Path B is a *forensic/observability* read surface (INV-ES-06 "why did this happen?"), **not** the state-reconstruction path ‚Äî Path A (re-derivation from `state_reported`, frozen AMD-50/AMD-51) remains the sole authoritative source of materialized state. So REC-104's value is "make trace reads of legacy rows lossless," not "change how state is rebuilt." The bit-determinism proof obligation (extend AMD-51 ¬ß5 #5b to *every* String‚Üítyped lift, golden-file corpus per Artium.AI/holixon) is accepted and is the verification gate. **This is Nick's scope call:** wire the decode-path lift now (REC-104) vs accept `DegradedEvent` for pre-AMD-52 rows and rely on Path A for all state. Closes G2's Path-B sub-fork once decided.

### REC-103 ‚Äî Confirm inline-BLOB with a constrained-hardware micro-benchmark; defer normalization permanently (Fork 5, gate G5-adjacent) ‚Äî **ACCEPT AS VALIDATION (not an authoring blocker)**
The inline-BLOB decision is already DECIDED+CLOSED (design beat G5). REC-103 is correctly framed as cheap validation, not a change ‚Äî and the evidence (SQLite fasterthanfs, the opaque-blob consensus, the deliberate Marten/Postgres-JSONB contrast) already confirms the choice on paper. **PM disposition:** AMD-52 authoring does **not** block on a Pi benchmark; the inline-BLOB shape is settled. Treat the micro-benchmark as an *optional M4.0b-x coding-instruction acceptance check* (append + full-stream replay at representative payload sizes; record GC under 256 MiB), not a gate. The SQLite 3.45 JSONB note is correctly scoped as a future opaque-compaction lever only ‚Äî out of scope for AMD-52.

### REC-105 ‚Äî AMD-52 anti-pattern gating checklist (Fork 6, process) ‚Äî **ACCEPT**
Adopt as-is into the AMD-52 author/coder gate. The 7 items are all real and each maps to a source war-story. **PM additions on adoption:** item 6 ("no `@JsonTypeInfo`/reflective polymorphism") must cite the **correct constraint per surface** ‚Äî see ¬ßF #3: ArchUnit Rule 7 covers `StateChangedEvent` (event package); the **Jackson-isolation HARD RULE** is what covers `AttributeValue` (device package).
