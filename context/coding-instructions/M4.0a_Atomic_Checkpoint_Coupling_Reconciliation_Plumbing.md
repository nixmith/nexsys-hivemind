## What This Implements

M4.0a lays the **data-integrity and reconciliation-metadata foundation** that the M4.0b backfill gate binds to. ### Dependency direction — load-bearing for the seam (deliverable 1)

The current `module-info.java` graph is **`com.homesynapse.persistence requires transitive com.homesynapse.state`** — persistence depends on state-store, **not** the reverse. `com.homesynapse.state` requires only `platform`, `device`, `event`, `event.bus`. Therefore:

- `StateProjection` (in `com.homesynapse.state`) **cannot** call a persistence type. Adding `requires com.homesynapse.persistence` to state-store would create a module cycle.
- The seam must be a **consumer-defined interface in state-store that persistence implements** — exactly the existing `StateCheckpointSource` / `ViewCheckpointStore` pattern (interface in state-store, `Sqlite*` implementation in persistence, injected through `PersistenceFactory` at the `HomeSynapseCore` composition root).
- AMD-45 §5's first bullet ("interface in state-store, implementation in persistence") is the correct direction; its second bullet ("expose through `PersistenceFactory` returning a persistence type") is rejected for state-store-facing wiring because it would invert the dependency. `PersistenceFactory` may still expose an accessor, but the **type `StateProjection` depends on must live in state-store (or platform)**.

## Behavioral Contracts (the eight deliverables)

### 1. Wire atomic checkpointing into the production path
Route `AtomicCheckpointWriter.writeAtomicCheckpoint(...)` into the production checkpoint path used by `StateProjection.writeCheckpoint()`. Today nothing in production calls it (only tests do); `StateProjection.writeCheckpoint()` writes only to `ViewCheckpointStore`.

**Call-seam constraint (do not hand-wave — see "Dependency direction" above):** `AtomicCheckpointWriter` is **package-private in `com.homesynapse.persistence`**; `StateProjection` is in `com.homesynapse.state` and cannot call it directly, and state-store cannot depend on persistence. Define the seam as a **state-store-side interface** (the `StateCheckpointSource` precedent — extend it or add a sibling such as `AtomicCheckpointSink`), implement it in persistence backed by `AtomicCheckpointWriter`, and inject it through `PersistenceFactory` at the `HomeSynapseCore` composition root. Respect JPMS (one flat package per module, no cross-module reflection) and the inward-only dependency direction. **The exact seam shape is yours to design against the dependency graph** — but it must not invert the dependency direction or leak a persistence type onto a state-store API surface.

### 2. Remove the per-delivery bus subscriber-checkpoint for `state_projection`
`InProcessEventBus` performs a per-delivery checkpoint write after every successful LIVE delivery (`checkpointStore.writeCheckpoint(subscriberId, envelope.globalPosition())`, ~line 500). For the `state_projection` subscriber this must be removed in favor of the atomic path.

**Chosen design — AMD-45 §2.2 Option A (Nick-confirmable):** opt `state_projection` out of per-delivery bus checkpointing via a new `SubscriberInfo.atomicCheckpoint` boolean. The existing `SubscriberInfo` already carries the `coalesceExempt` boolean (the coalesce-exempt pattern), so the per-subscriber-flag pattern is established — `atomicCheckpoint` mirrors it. The bus skips the per-delivery write when `atomicCheckpoint == true`. Option B (a no-op `CheckpointStore` wrapper) is **rejected** as less explicit. Implement Option A.

### 3. Fold OR-M3-13 — populate the reconciliation metadata (M4.0b's backfill gate binds to this)
`reconciledAt` / `reconciledFromVersion` / `reconciledToVersion` **already exist** in `CheckpointData` and are already (de)serialized by `CheckpointSerializer`, but are written **`null`** today: `SqliteStateStore.serializeCheckpoint(int)` calls `serializer.serialize(snapshot, projectionVersion, null, null, null)`, and `StateCheckpointSource.serializeCheckpoint(int)` accepts only `projectionVersion`.

**The work is population of existing fields via an extended `serializeCheckpoint` surface** — not invention of new fields. Extend the `serializeCheckpoint` interface/path so the projection *populates* these fields when a version transition occurs (it sets `reconciledFromVersion`/`reconciledToVersion` on the reconciliation triggered by `persistedVersion != projectionVersion` — e.g. the 1→2 transition M4.0b will trigger; `reconciledAt` from the injected `Clock`). **`reconciledToVersion` is a downstream dependency: M4.0b's backfill gate binds to it. It is not optional polish.** This un-defers `ReconciliationTest`'s 5th test (`reconciliationRecordsMetadataInDataSlot`), which M3.6d-a deferred because it would have failed trivially against the null contract.

### 4. Fold H2 — the dedup transaction wrapper
Implement the H2 item (`AtomicCheckpointWriter` two-way / three-way methods duplicate their transaction wrapper; extract a shared `executeInTransaction`-style helper) **within the atomic-write transaction boundary**. Reference it as plan item **H2** (from M3.5b Cowork review, tracked in pm-handoff Open Risks). Scope it to the atomic-write path; do not refactor unrelated transaction sites.

### 5. `HomeSynapseCore` Javadoc fix (two corrections in one edit)
The Javadoc calls the flow "state_reported → state map update" — imprecise: on `state_reported` the `EntityState` record is replaced (version/timestamps advance) but `attributes` are **not** touched; only `state_changed` updates `attributes`. Correct that wording. **In the same edit**, the same Javadoc block calls the `state_changed` consumer (the automation engine) **"M5 scope"** — it is **M7/M8** (M5 is Platform API). Fix **"M5" → "M7"**. (Both corrections are in the class Javadoc near the `MINIMAL_DERIVATION_RULE` documentation — confirm exact lines at the STOP gate.)

### 6. REC-80 — replay-duration metric
Emit `projection.replay.duration_ms` and `events_replayed` at the end of a reconciliation / first-boot replay. Pure addition. **No snapshot-store code** — `SqliteSnapshotStore` stays dormant until a *measured* p95 full-replay on Pi exceeds the 5 s AMD-41 §3.2.3 threshold; this metric is what later feeds that decision. Confirm whether REPLAY exits through `onEvent` or `processBatch` before placing the metric (STOP gate).

### 7. REC-82 — guard the `projectionVersion` footgun
Reconciliation compares `checkpointSource.loadedProjectionVersion()` against the code's `projectionVersion` — **not** `CheckpointRecord.projectionVersion()`, which is a sentinel hardcoded to `1` by the view-checkpoint store implementations. Deprecate/guard `CheckpointRecord.projectionVersion()` (point callers to `loadedProjectionVersion()` as authoritative — Javadoc + `@Deprecated` or equivalent guard) and add a CI regression test asserting reconciliation reads the **loaded** value, so a future caller cannot silently bind to the sentinel.

### 8. Extend `CrashRecoveryHttpIT` — reproduce AMD-45 §3's concrete scenario
Assert the bus-ahead-of-view window is closed by reproducing AMD-45 §3's exact regression — an abstract "window closed" assertion is insufficient because it can pass without exercising the no-checkpoint-fired replay-from-zero case that is the whole point of the fix. The test must:

1. Configure with `HOME_DEFAULT` (200-event / 2-second policy) — **not** `TESTING` (N=1 would let a checkpoint fire and mask the bug).
2. Publish N events where N < 200 (e.g. 5 entities), reach LIVE, and verify all N entities are queryable via HTTP.
3. Call `abandon()` **before any checkpoint has fired** (N < 200 events *and* the policy's time threshold not yet crossed — see the clock note below).
4. Restart on the same database path; reach LIVE.
5. Assert all N entities are queryable via HTTP.

Before AMD-45 this fails at step 5 (bus subscriber checkpoint at position N causes replay to be skipped, no view checkpoint exists, state lost). After AMD-45 it passes (coupled checkpoint never advanced, so the bus replays from 0 and the projection rebuilds all N). Route the crash through `HomeSynapseE2eHarness.abandon()` (the production-grade path) — **do not** use the flag-based `IntegrationTestHarness.abandon()` path, and do not unify the two (see the two-abandon-paths gotcha).

**Clock:** make an **injected sub-threshold `Clock` the primary mechanism** — inject a `Clock` that stays below the 2 s policy threshold so the time branch cannot fire (AMD-45 §3's stated fallback, promoted to primary here). With N < 200 the event-count threshold also cannot fire, so the no-checkpoint scenario is reproduced deterministically with **no** `Clock.systemUTC()` and **no** `NO_DIRECT_TIME_ACCESS` whitelist/config change. A real-clock variant is acceptable only as a secondary path if the injected approach proves insufficient — not the reverse.

**Test posture — contract-test-first; extend, do not greenfield.** `ReconciliationTest` exists — extend it (the OR-M3-13 5th test lands here). `CrashRecoveryHttpIT` exists — extend it.

## What to Watch Out For

- **Seam direction (the big one).** state-store must not gain `requires com.homesynapse.persistence`. Mirror `StateCheckpointSource`: interface in state-store, `Sqlite*`/persistence implementation, `PersistenceFactory` accessor, composition-root injection in `HomeSynapseCore`. If you find yourself adding a persistence import to `StateProjection`, stop — that is the wrong direction.
- **Virtual threads + `synchronized` (LTD-11).** The atomic write runs through the persistence write executor (platform thread, AMD-26/27). Use `ReentrantLock`, never `synchronized`, anywhere on the path — including any helper you extract for H2.
- **`SubscriberInfo` is a `public record`.** Adding `atomicCheckpoint` changes its canonical constructor arity. Grep for shape tests (`SubscriberInfoTest`) and every construction site (composition root + testFixtures) and update them; specify a sensible default (existing subscribers = `false`).
- **Idempotency is load-bearing (INV-ES-05).** Option A makes the `state_projection` subscriber checkpoint advance on policy cadence, so crash recovery replays up to `eventThreshold` events. `StateProjection.onEvent` → `StateStore.put()` is naturally idempotent; the `MINIMAL_DERIVATION_RULE` lambda derives nothing, so it is trivially idempotent. Do not add state that breaks this.
- **Two abandon paths (do not unify).** HTTP-aware crash assertions go through `HomeSynapseE2eHarness.abandon()`; the flag-based `IntegrationTestHarness.abandon()` backs the passive M3.4 harness. The difference is intentional until Doc 15 Layer 2.
- **`@TempDir` on Windows.** `CrashRecoveryHttpIT` uses `CleanupMode.NEVER` because the abandoned harness holds SQLite handles. Preserve this when extending.
- **Jackson isolation.** Reconciliation metadata serializes through `CheckpointSerializer` (already Jackson-isolated in persistence). Do not let any Jackson type leak onto a public API surface, and do not introduce a new ObjectMapper — reuse the existing path.
- **Clock injection in tests — no exception.** `NO_DIRECT_TIME_ACCESS` fails the build on `Instant.now()`/`System.nanoTime()`/`System.currentTimeMillis()` in non-whitelisted test code. Inject `Clock.fixed(...)` everywhere, including deliverable 8: an injected sub-2 s `Clock` reproduces the no-checkpoint-fired scenario deterministically (N < 200 already prevents the event-count threshold from firing), so `Clock.systemUTC()` is not needed and the ArchUnit rule is never tripped.
- **`reconciledToVersion` must be populated, not just plumbed.** A test that only checks the field is *serializable* is insufficient; assert it carries the transition's target version after a version-mismatch reconciliation.
