## What This Implements

AMD-51 (shipped `98f705b`) made change-detection typed but deliberately kept the `StateChangedEvent` payload as `String` and materialized `new StringValue(...)` — the typed values it reconstructs for comparison are thrown away. M4.0b-4 ends that staging: it carries the typed `AttributeValue` into the event payload (S1) and into the materialized state + checkpoint (S2). Concretely, `ProductionDerivationRule` already computes `inboundTyped`/`priorTyped` for the AMD-51 comparison — this WU emits *those* values in a typed `StateChangedEvent` (written at per-event `schema_version = 2`) instead of stringifying them, `StateProjection.applyToState` materializes the typed value, `CheckpointSerializer` serializes a typed envelope, and a new custom `AttributeValue` Jackson codec in `core/persistence` (de)serializes the 8-variant sealed type with an explicit `AttributeType` tag (no `@JsonTypeInfo`). The materialized-output change forces `projectionVersion` 3→4, riding AMD-50's frozen reconciliation-backfill unchanged. Historical `schema_version = 1` String payloads are never read for state (Path A re-derives from the immutable `state_reported` log); a forensic typed read of one degrades to a `DegradedEvent` (Path B, raw preserved).

This is the single most cross-cutting change in Workstream A — it touches the serializer, the event-store payload shape, and replay determinism at once. Read AMD-52 in full before starting.

## Settled Decisions (from AMD-52 — NOT open questions)

- **DP-1 (envelope).** Compact tagged-union `{"t":"<AttributeType.name()>","v":<value>}`. `"t"` = `attributeType().name()` (plain string field, NOT `@JsonTypeInfo`). Scalars: `BooleanValue`→bool, `IntValue`→number(long), `FloatValue`→number or sentinel string (DP-3), `StringValue`/`EnumValue`→string. `QuantityValue`→`"v"` = canonical magnitude (DP-3 float rules), plus `"u"` = canonical unit symbol. `ArrayValue`→`"v"` is a JSON array; **recurse the same envelope per element, in order**. `DegradedAttributeValue`→`originalTypeName`/`rawForm`/`failureReason` fields. (AMD-52-INV-02)
- **DP-2 (no reflection / Jackson isolation).** Hand-rolled serde, no `@JsonTypeInfo`, no Jackson annotation on `AttributeValue`/`StateChangedEvent`/any event or device type. Serde lives only in `core/persistence`. Dispatch = exhaustive `switch` over the sealed type, **no `default`** arm. (AMD-52-INV-02; AMD-51-INV-01 twin)
- **DP-3 (float identity + edges).** Render finite doubles with a **round-trippable** representation (`parseDouble(render(x))` recovers the same bits) — do NOT freeze a canonical text renderer; `Double.toString`/Jackson default is acceptable *as long as round-trip holds*. Identity is the **bits** (`Double.doubleToLongBits` after `−0.0`→`+0.0` + canonical-NaN), never the text. `NaN`→`"NaN"`, `+∞`→`"+Inf"`, `−∞`→`"-Inf"` sentinel strings; strict decoder rejects unknown tokens; `−0.0`→serialize as `+0.0`. `ALLOW_NON_NUMERIC_NUMBERS` stays disabled. `chain_hash` is NOT touched (stays AMD-37 zero-reservation). (AMD-52-INV-03/-04)
- **DP-4 (schema_version 1→2).** The typed `StateChangedEvent` draft is emitted with `schemaVersion = 2`. No new column, no migration. (AMD-52-INV-01)
- **DP-5 (Path A authoritative; Path B = DegradedEvent).** The 3→4 backfill re-derives typed state from `state_reported` (Path A, unchanged AMD-50/AMD-51). A typed read of a legacy `schema_version == 1` `state_changed` returns a `DegradedEvent` (raw preserved). The version gate lives in `EventPayloadCodec.decode`. **Do NOT** wire `AttributeValueUpcaster` or any schema resolver into the codec/decode path. (AMD-52-INV-05)
- **DP-6 (oldValue nullable).** `StateChangedEvent.oldValue` is nullable; `null` = first report (replaces today's `""` sentinel). `newValue` always non-null. Both consumers already special-case first-report. (AMD-52 §2.1)
- **DP-7 (projectionVersion 3→4).** Bump at the composition root; AMD-50 reconciliation does the rest. Do NOT author a new backfill. (AMD-52-INV-07)

## Technical Specification

### `AttributeValueSerializer` (class extends `com.fasterxml.jackson.databind.JsonSerializer<AttributeValue>`)
**Package:** `com.homesynapse.persistence` (package-private). **Mirror `UlidSerializer`.**

```java
@Override public void serialize(AttributeValue v, JsonGenerator gen, SerializerProvider p) throws IOException;
@Override public Class<AttributeValue> handledType();   // AttributeValue.class
```
**Behavioral contract:** writes a single object `{ "t": <attributeType().name()>, "v": <variant payload> [, "u": <unit>] }`. Dispatch via an **exhaustive `switch`** over the sealed `AttributeValue` with **no `default`** (a 9th permit must break compilation). `ArrayValue` writes `"v"` as an array, recursing `serialize` per element in list order. `FloatValue`/`QuantityValue` magnitude via a shared package-private `static void writeDouble(JsonGenerator, double)` that emits a sentinel string for `NaN`/`±Inf` and writes `+0.0` for `−0.0`, else a round-trippable number. Fields written in fixed order. Pure; thread-safe (stateless).

### `AttributeValueDeserializer` (class extends `JsonDeserializer<AttributeValue>`)
**Package:** `com.homesynapse.persistence` (package-private). **Mirror `TypedUlidDeserializer`.**

```java
@Override public AttributeValue deserialize(JsonParser pp, DeserializationContext ctx) throws IOException;
```
**Behavioral contract:** reads `"t"`, maps to `AttributeType`, dispatches to the variant constructor. A shared `static double readDouble(JsonNode/JsonToken)` reverses the sentinels (strict — an unknown string token throws `JsonMappingException`). `ArrayValue` recurses per element. An unknown/missing `"t"`, a malformed `"v"`, or a sentinel-decode failure throws (caught by `EventPayloadCodec` stage-2 → `DegradedEvent`; or by `CheckpointSerializer.deserialize` → `IllegalStateException` → reconciliation). **Do NOT** consult any `AttributeValueUpcaster` here.

## What to Watch Out For

- **`Clock` in tests (ARCH-RULE, will fail `./gradlew check`).** Both `com.homesynapse.persistence` and `com.homesynapse.state` are **non-whitelisted** for `NO_DIRECT_TIME_ACCESS`. Do NOT use `Clock.systemUTC()`, `Instant.now()`, `System.nanoTime()`, or `System.currentTimeMillis()` anywhere in test code. Use `Clock.fixed(Instant.parse("2026-01-01T00:00:00Z"), ZoneOffset.UTC)` and inject it via constructor/`@BeforeEach`. (M2.4/M2.5 both tripped this — see `coder-lessons.md` 2026-04-10.)
- **The `−0.0`/NaN canonicalization must match the comparator.** AMD-51 §2.3 maps `−0.0`→`+0.0` and treats `NaN`↔`NaN` as unchanged. The serializer must canonicalize identically (store `+0.0`, single `"NaN"` sentinel) so a value the comparator calls "unchanged" never serializes to two different forms. A single `"NaN"` sentinel collapses signaling/quiet NaNs — that is the intended, ratified behavior (AMD-52 §2.3 / Research 11 Caveat 3).
- **`NON_NULL` vs the checkpoint.** The shared `PersistenceObjectMapper` is `Include.NON_NULL`; `CheckpointSerializer` needs nulls preserved. Keep its `ALWAYS`-mapper requirement intact — do not let the typed envelope start dropping null attribute values or null `staleAfter`. Verify the round-trip test covers both.
- **`Include.NON_NULL` and the envelope.** On the event-payload mapper, `NON_NULL` will omit a null `oldValue` field — that is correct and desired (first-report). Make sure the deserializer treats an absent `old_value` as `null`, not an error (the record allows null `oldValue`).
- **Field naming / `SNAKE_CASE`.** The mapper is `SNAKE_CASE`. `StateChangedEvent` fields serialize as `attribute_key`/`old_value`/`new_value`/`triggered_by`. Inside the hand-rolled `AttributeValue` codec you control the field names directly (`"t"`/`"v"`/`"u"`) — `SNAKE_CASE` does not rewrite literals you write via `gen.writeFieldName`, but do not rely on it; write them exactly.
- **Event-model → device-model module edge (the likely STOP).** See the PM note under `StateChangedEvent`. Most other "no new module edge" claims are verified; this one is not. Report before adding.
- **`ArrayValue` element type on decode.** The envelope is self-describing (each element carries its own `"t"`), so the deserializer does NOT need element-schema metadata — recurse on the per-element `"t"`. (This is why the codec is self-sufficient where AMD-51's schema-driven reconstruction needed a schema.)
- **Do not touch `chain_hash`, the AMD-50 backfill, the `AttributeValueUpcaster` SPI, or `DerivationContext` (no clock).** All frozen by AMD-52 §6.
- **Both projection paths.** `applyToState` is reached on both `onEvent` and the batch path; the typed write must hold on both, and the backfill twin (`applyBackfillAttribute`) must write typed too (AMD-47-INV-02 both-paths discipline; AMD-50 backfill).
