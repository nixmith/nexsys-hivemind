## What This Implements

A **pure, behavior-preserving** module/package relocation: move the self-contained `AttributeValue` value hierarchy out of `com.homesynapse.device` into a new leaf module `com.homesynapse.value` that both event-model and device-model depend on. This breaks the event↔device JPMS cycle that AMD-52's typed `StateChangedEvent` payload would create (design-note §1) and unblocks M4.0b-4b. **No semantics change, no `projectionVersion` change, no behavioral contract change** — the value types' records, fields, sealing, canonicalization, and `AttributeType` values are byte-for-byte the same; only their `package` and owning module change.

## Technical Specification

This is a relocation, not new logic. The only "spec" is: **the value types' public API is identical after the move** (same record components, same `sealed`/`permits`, same `QuantityValue` canonicalize-at-construction, same `AttributeType` constants and order, same Javadoc) — only `package`/module change. The `requires` rule (design-note §4): a module that **re-exports** value types on its own public API uses `requires transitive com.homesynapse.value`; a module that only **consumes** them internally uses plain `requires com.homesynapse.value`. The spike confirmed the exact set — follow it.

## What to Watch Out For

- **Device-model in-package users (the easy miss).** Files in `com.homesynapse.device` that referenced `AttributeValue`/`AttributeType`/a variant with **no import** (same package) will now fail to compile until you add `import com.homesynapse.value.…`. `AttributeSchema` and `AttributeValueUpcaster` are the prime cases — they stay in device-model and must import from value. Grep device-model for bare uses, not just `import` lines.
- **`requires transitive` vs plain.** Get this right or downstream modules silently lose visibility: event-model and device-model re-export the value types (their public APIs mention them), so **transitive**; persistence consumes `AttributeValue` internally in the codec/`CheckpointSerializer` (does not re-export), so **plain**. The spike's findings are authoritative.
- **Do NOT touch the vestigial `device → event` edge as part of this WU** unless the spike proved it dead AND removing it is trivial — and **never** invert to `event → device` (design-note §7; wrong layering, re-breaks later). If `device → event` is dead, dropping it is optional hygiene; flag it, do not bundle risk.
- **No semantics.** Do not change `projectionVersion`, `StateChangedEvent` (still String payload — that is 4b), any comparator/reconstructor logic, the codec (does not exist yet — 4b), or any test assertion's expected values. If a test breaks on anything other than an import path, STOP and report — the move is not behavior-preserving and something is wrong.
- **`AttributeValue` carries no Jackson annotation** before or after (Jackson-isolation HARD RULE) — the move does not change that; the codec that serializes it is 4b, in persistence.
- **Test-`Clock` arch-rule (§4c).** You are unlikely to add test code here, but if you adjust any test in `com.homesynapse.{persistence,state}` (non-whitelisted), do NOT introduce `Instant.now()`/`Clock.systemUTC()`/`System.nanoTime()`/`currentTimeMillis()` — inject `Clock.fixed(...)`. `NO_DIRECT_TIME_ACCESS` will fail `./gradlew check`.
