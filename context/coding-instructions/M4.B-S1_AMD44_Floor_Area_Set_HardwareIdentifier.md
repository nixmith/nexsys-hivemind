## What This Implements

HomeSynapse's spatial model is flat today: `Device`/`Entity` carry an `AreaId`, but nothing groups areas into floors, so a multi-story home cannot express "all lights on the ground floor" as one selector. AMD-44 Stage 1 introduces the **Floor aggregate** (the `FloorId` typed ULID, the `Floor` record, the `FloorRegistry` interface), a **minimal `Area` record + read-only `AreaRegistry`** (so `Area.floorId` is structurally expressible), corrects the `AreaId` Javadoc that conflates area and floor, and fixes a latent data-quality bug by changing `hardwareIdentifiers` from `List` to **`Set`** (the value-record `HardwareIdentifier` has meaningless duplicates). Stage 1 is a self-contained structural change — new types with no existing consumers, plus a mechanical collection-type refactor — with the blast radius limited to `device-model` and its direct dependents. **EntityRole and everything that touches `EntityType`/`Entity`/`ProposedEntity` is Stage 2 and is explicitly out of scope here.**

This milestone is **interface- and record-level**, matching the current device-model phase: the existing registries (`DeviceRegistry`, `EntityRegistry`, `CapabilityRegistry`) are **interfaces with no production implementation** in this module. `FloorRegistry` and `AreaRegistry` are authored the same way — **interfaces only**. No registry implementation, no event emission, no first-boot synthetic-Area logic ships in Stage 1 (those have no home until the registries are implemented; see Out of Scope).

## Technical Specification

### `FloorId` (record) — `com.homesynapse.platform.identity`
**Responsibility:** Typed ULID wrapper for a floor, mirroring `AreaId`/`DeviceId`/`EntityId`.
**Implements:** `Comparable<FloorId>`.

```java
public record FloorId(Ulid value) implements Comparable<FloorId> {
    public FloorId { Objects.requireNonNull(value, "FloorId value must not be null"); }
    public static FloorId of(Ulid value) { return new FloorId(value); }
    public static FloorId parse(String crockford) { return new FloorId(Ulid.parse(crockford)); }
    @Override public int compareTo(FloorId other) { return value.compareTo(other.value); }
    @Override public String toString() { return value.toString(); }
}
```
Full Javadoc on the record + each static method, matching `AreaId`'s style. Storage is `BLOB(16)` per LTD-04 (string conversion only at API/log boundaries — not this module's concern). **No Jackson (de)serializer in this module** — domain/identity types stay Jackson-free (AMD-52-INV-02 / `NO_JACKSON_IN_DOMAIN_MODEL`); any `FloorId` serde lives in persistence/rest-api as a later WU.

### `Floor` (record) — `com.homesynapse.device`
**Responsibility:** The floor aggregate (AMD-44 §2.1.2).

```java
public record Floor(
        FloorId id,           // never null
        String name,          // non-blank, <= 100 chars
        int level,            // signed: -1 basement, 0 ground, 1 first, …
        String icon,          // nullable — MDI icon name, e.g. "mdi:home-floor-g"
        List<String> aliases, // voice synonyms; unmodifiable via List.copyOf()
        Instant createdAt     // never null
) {
    public Floor {
        Objects.requireNonNull(id, "Floor id must not be null");
        Objects.requireNonNull(name, "Floor name must not be null");
        Objects.requireNonNull(createdAt, "Floor createdAt must not be null");
        if (name.isBlank()) throw new IllegalArgumentException("Floor name must not be blank");
        if (name.length() > 100) throw new IllegalArgumentException("Floor name must be <= 100 chars, got " + name.length());
        aliases = List.copyOf(aliases); // NPE on null elements/collection — matches project convention
    }
}
```
**Contracts:** `icon` is nullable `String` (NOT `Optional` — Decision 7); no uniqueness constraint on `level` (split-level homes — Decision 8); `aliases` defensively copied. Immutable.

### `FloorRegistry` (interface) — `com.homesynapse.device`
**Responsibility:** Floor lifecycle surface (AMD-44 §2.1.3). **Interface only — no implementation in this WU** (matches `DeviceRegistry`/`EntityRegistry`).

```java
public interface FloorRegistry {
    Floor create(String name, int level, String icon, List<String> aliases);
    Optional<Floor> get(FloorId id);
    Collection<Floor> getAll();          // sorted by level ASC, then name ASC, then createdAt ASC
    Collection<Floor> getByLevel(int level);
    Floor update(FloorId id, String name, int level, String icon, List<String> aliases);
    void delete(FloorId id);             // contract: rejects if areas are still assigned (cascade is REST/impl concern, Decision 11)
}
```
Javadoc must state: `getAll()` ordering (`level ASC, name ASC, createdAt ASC` — Decision 8); `delete` contract (rejects when areas remain assigned — the cascade/`?force=true` behavior is the REST/impl layer's concern, not declared here); implementations must be safe for concurrent access and use `ReentrantLock`, never `synchronized` (LTD-11) — stated for the future implementer.
