## What to Watch Out For

- **Tests must inject `Clock`.** Do NOT use `Clock.systemUTC()`, `Instant.now()`, `System.nanoTime()`, or `System.currentTimeMillis()` anywhere in test code. The ArchUnit rule `NO_DIRECT_TIME_ACCESS` scans test classes in non-whitelisted packages (integration + event + config + device are ALL non-whitelisted) and will fail `./gradlew check`. Use literal `Instant.parse("2026-01-01T00:00:00Z")` for record components (the M4.B-S2 pattern) or `Clock.fixed(...)`. This reminder exists because M2.4 and M2.5 both tripped this rule; see coder-lessons.md 2026-04-10.
- **The record-evolution trap:** adding components to `IntegrationDescriptor`/`IntegrationContext` breaks every positional constructor caller. The convenience ctors exist precisely so `StubIntegrationContext`, `TestAdapter` flows, and all existing tests compile **unchanged** — verify zero edits needed in callers you didn't list (the B-S2 "zero existing-caller breakage" bar).
- **`IntegrationEvents` Javadoc** says "Adding a new lifecycle event subtype requires editing this list (the forcing function)" — when you edit the list, also re-read `AllEventClasses.java` (persistence test) which concatenates it; the 5→10 change flows through automatically, but the test's documented counts may pin 5.
- **EventTypes constants live in event-model** — do not put the new strings in integration-api (the existing 5 INTEGRATION_* constants set the precedent). String constants only; no event-model records.
- **`CapabilityAdded` serde is the riskiest item:** generic Jackson serialization of the `CapabilityInstance` subtree (`AttributeSchema`/`CommandDefinition`/`ConfirmationPolicy` + nested maps/lists). If any nested type fails round-trip without annotations or new mixins, **STOP and report** — do not add Jackson annotations to domain types (ArchUnit will fail) and do not silently write custom serializers (that is an AMD-59 design question for the PM).
- **`previousState` nullability:** the sealed parent documents `previousState()` nullable **only for IntegrationStarted** — the five new permits carry non-null current state in both fields; null-guard accordingly (do not copy IntegrationStarted's null-allowance).
- **Do not reorder enum constants** (`RequiredService`, `ExceptionClassification` append-only — declaration order is pinned by tests, the AMD-44/B-S2 source-order lesson).
- **Sandbox `git`/`grep` distrust:** use the Read/file tools for every verification; the bash mount has served truncated copies repeatedly (standing lesson).
