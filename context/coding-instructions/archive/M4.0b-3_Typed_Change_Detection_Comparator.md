## What This Implements

The State Projection's production change-detect rule (`ProductionDerivationRule`) currently decides whether an inbound `state_reported` is a change by **string comparison** (`Objects.equals` over stringified values). That conflates `21.0`/`21.00`, can't reason about float FP-noise, quantity dimensional equality, array structure, or the degraded sentinel — producing phantom-change event storms. AMD-51 replaces the string compare with a **typed** comparison: reconstruct both operands to their schema-declared `AttributeValue` variant, then compare per-variant (exact for Boolean/Int/Enum/String; pinned total-form epsilon for Float/Quantity; order-sensitive deep compare for Array; the HA-mirrored rule for Degraded). The comparator is an external, pure, total `AttributeValueComparator` in state-store carrying a `ComparisonPolicy`. The **String `StateChangedEvent` payload is preserved** (typed payload is AMD-52, deliberately staged). Because typed compare changes change-detection outcomes, this is a derivation-logic change that **bumps `projectionVersion` 2→3** and rides AMD-50's reconciliation-backfill for the transition.

You are implementing the comparator, the `ComparisonPolicy`, the symmetric schema-driven reconstruction step, the wiring into `ProductionDerivationRule` + the composition root, and the full test suite — **tests first**.

## Settled Decisions (from AMD-51 — NOT open questions)

Implement exactly as stated; these are ratified.

- **DP-A — Comparator placement & shape (AMD-51-INV-04).** An external `AttributeValueComparator` interface in `com.homesynapse.state`, package-private impl `StructuralAttributeValueComparator`, reached via a **public static factory** (DEC-M3-16 gateway, mirroring `DerivationRule.production()` / `StateQueryService.materialized()` / `StateCheckpointSource.stub()`). NOT a method on `AttributeValue`.
- **DP-B — Total, exhaustive, no `default` (AMD-51-INV-01).** The per-variant dispatch is an exhaustive `switch` over the sealed `AttributeValue` (8 arms) with **no `default`**. A future 9th permit must break compilation. (D-01 is event-type-scoped; this is permitted.)
- **DP-C — Emit predicate, not equality.** The method answers "should a `state_changed` be emitted?" — folding the asymmetric Degraded rule and the `prior == null` first-report case. Name it to signal this (`changed` / `hasChanged` / `shouldEmitChange` — never `isEqual`/`compare`); Javadoc the asymmetry. NOT symmetric equality.
- **DP-D — Pinned epsilon (AMD-51-INV-02).** `changed ⟺ |a−b| > max(absEps, relEps·max(|a|,|b|))`, defaults `absEps = relEps = 1e-9`, in `ComparisonPolicy`. IEEE-754 totality exactly: `NaN`↔number = changed; `NaN`↔`NaN` = unchanged; `+0.0`↔`−0.0` = unchanged; same-sign `Inf` = unchanged; opposite-sign or finite↔`Inf` = changed. **`Inf−Inf = NaN`, so handle the same-sign-Inf case BEFORE the arithmetic** (do not let the formula see `NaN`). Pure relative epsilon is rejected.
- **DP-E — Quantity units are free.** `QuantityValue` is canonical at construction (AMD-47-INV-03). Quantity↔Quantity: if canonical `unit()` symbols differ ⇒ changed (different dimension); else apply DP-D epsilon to `value()`. **No unit lookup/conversion in the comparator.** (`QuantityValue` can't carry NaN/Inf — keep the totality checks as defence-in-depth with a comment, per AMD-51 §10.)
- **DP-F — Degraded (AMD-51-INV-03).** Inbound Degraded ⇒ never emit; prior Degraded + valid inbound ⇒ emit (recovery); two Degraded ⇒ unchanged.
- **DP-G — String payload preserved (AMD-51 §2.7).** When `changed == true`, build the **same** `StateChangedEvent(key, oldStringForm, newValue, env.eventId())` the rule builds today (`oldStringForm` = stringified prior via the existing `priorValue` extraction pattern; `newValue` = `sr.value()`). Do NOT make the payload typed. Do NOT touch `CheckpointSerializer` or the event-store shape (AMD-52 territory).
- **DP-H — Symmetric reconstruction (AMD-51-INV-05).** **Both** operands are reconstructed to the schema-declared typed variant before the comparator runs: the inbound `StateReportedEvent.value` (+ `unit` for QUANTITY, fallback `AttributeSchema.canonicalUnitSymbol`) **and** the prior — which is always a `StringValue` (or null), so its `value()` is reconstructed by the identical parse. Reconstruction is keyed by `AttributeSchema.type`, is **distinct from** the `AttributeValueUpcaster` SPI (leave the SPI unchanged), and the produced typed values are **transient** (the stored attribute + emitted payload stay String, DP-G).
- **DP-I — `projectionVersion` 2→3.** Bump the literal at `HomeSynapseCore`. The transition rides AMD-50's reconciliation-backfill **unchanged** (AMD-50 §2.5 / scenario 3.3). The new wrinkle is string-history → typed reconstruction during the 2→3 backfill, which uses the **same** reconstruction step as LIVE (determinism).
- **DP-J — Deadband deferred (REC-95).** Do NOT implement a deadband. `ComparisonPolicy` may reserve the shape but carries no deadband field.
- **DP-K — Schema source (PRE-DECIDED — resolves the former Key Design Decision; verified against source 2026-05-30).** There is **no** production `CapabilityRegistry`/`EntityRegistry` implementation and **no** entity→capability binding (verified). The schema source for reconstruction is therefore built from the **standard capability catalog**: create a production **`StandardCapabilities`** factory in `com.homesynapse.device` (device-model **main**, NOT testFixtures) that returns the 15 standard capability instances with their correct `attributeSchemas()`, lifting the construction logic from `core/device-model/src/testFixtures/.../TestCapabilityFactory.java` (verified self-contained: it imports only production `com.homesynapse.device.*` types + `java.time`/`java.util`, no test-only deps, no mocking). `HomeSynapseCore` calls `StandardCapabilities.all()`, aggregates every capability's `attributeSchemas()` into a single immutable `Map<String, AttributeSchema>` keyed by `attributeKey`, and builds the `AttributeSchemaResolver` as a pure `Map::get` lookup. This is **immutable compile-time-shaped config, morally identical to `QuantityValue.CATALOGUE`** — NOT a runtime registry read — so it satisfies **AMD-50-INV-03** (the rule reads injected immutable config, not a registry). The `TestCapabilityFactory` SHOULD be refactored to delegate to `StandardCapabilities` to avoid duplication (flag as a deviation if you do). **Scope note:** this lifts test-fixture construction into production — a deliberate, bounded ~1-class scope addition that is load-bearing (without it the comparator is inert and the 2→3 version bump is wasted) and forward-useful (the future `CapabilityRegistry` impl will seed standard capabilities from it).
  - **Resolver key is `attributeKey` only, not `(EntityId, attributeKey)`** — there is no entity→capability binding yet. **Assumption (documented, holds for the standard catalog — verified no key collides with a differing type; `power_w` is FLOAT in both PowerMeasurement and PowerMeter):** `attributeKey → AttributeType` is globally consistent across the standard capabilities. ## Technical Specification

### `AttributeValueComparator` (interface) + `StructuralAttributeValueComparator` (package-private impl)
**Package:** `com.homesynapse.state`
**Responsibility:** decide, per AMD-51 §2.2/§2.4b, whether a reconstructed inbound value represents a change that should be emitted, relative to the (reconstructed) prior value, under a `ComparisonPolicy`.

```java
public interface AttributeValueComparator {
    /**
     * Returns true iff a state_changed should be emitted for this attribute, given the
     * prior canonical value (nullable — null means first report) and the reconstructed
     * inbound value, under {@code policy}. Total over the 8-variant AttributeValue
     * hierarchy. Pure: no clock, no I/O, no randomness, no registry. Emit-predicate,
     * NOT symmetric equality (folds the Degraded asymmetry and the null-prior case).
     */
    boolean changed(AttributeValue prior, AttributeValue inbound, ComparisonPolicy policy);

    static AttributeValueComparator structural() { return new StructuralAttributeValueComparator(); }
}
```

**Behavioral contracts:**
- Exhaustive `switch` over `inbound` (and `prior`) variants, **no `default`** (DP-B). Type-mismatch between prior and inbound (non-Degraded) ⇒ changed. `prior == null` ⇒ changed unless inbound Degraded (DP-F).
- Float/Quantity epsilon per DP-D/DP-E. Array: size-then-order-sensitive deep recursion reusing `policy` (DP-B). Exact `.equals` for Boolean/Int/Enum/String.
- Pure & deterministic (AMD-50-INV-03). Thread-safe (stateless; safe to share).

### `ComparisonPolicy` (record)
```java
public record ComparisonPolicy(double absEps, double relEps) {
    public ComparisonPolicy { /* require finite, >= 0 */ }
    public static final ComparisonPolicy FP_NOISE_DEFAULT = new ComparisonPolicy(1e-9, 1e-9);
}
```
Reserve (comment only) the future per-attribute deadband home (DP-J) — no field now.

### `AttributeValueReconstructor` (pure helper) + `AttributeSchemaResolver` (seam)
`reconstruct(String serializedValue, String unitOrNull, AttributeSchema schema) → AttributeValue`, keyed by `schema.type()`:
- BOOLEAN/INT/FLOAT/STRING/ENUM → parse into the variant.
- QUANTITY → `new QuantityValue(parseDouble(serializedValue), unit)` where `unit = unitOrNull` if present/recognised else `schema.canonicalUnitSymbol()`; ctor canonicalizes (identity when already canonical). **Log a WARNING (attributeKey + entity id) when the `canonicalUnitSymbol` fallback is used** (silent-corruption guard, AMD-51 §2.6). If neither yields a recognised unit / parse fails → strict halt or Degraded per AMD-47-INV-04 (a Degraded value is never written to canonical state; DP-F suppresses an inbound Degraded).
- ARRAY → reconstruct elements per the element type (full-replacement).
Pure; reads only the injected immutable schema snapshot (determinism — see "Key Design Decision").

### Error Handling
| Condition | Behavior | Notes |
|---|---|---|
| Inbound value unparseable for `schema.type()` | strict halt OR Degraded inbound (→ DP-F never-emit) | Match AMD-47-INV-02/-04; no Degraded to canonical state. |
| QUANTITY `unit` absent/blank | fall back to `canonicalUnitSymbol`, **WARN** | If still unrecognised → Degraded path. |
| `schema` not found for (entity, key) | reconstruction cannot proceed → Degraded / no-emit | Surface via log; do not throw past the rule. |
| Future `AttributeValue` permit added | **compile error** (no `default`) | This is the desired AMD-51-INV-01 behavior. |

Error/log messages: Register C voice (direct, neutral, no "we"/"sorry"/"please").

## What to Watch Out For

1. **The prior side is `StringValue`, not the schema variant** (source-confirmed: `applyToState` L819 + `applyBackfillAttribute` L928 write `new StringValue(...)`; `CheckpointSerializer` "only writes StringValue"). A comparator that compares prior `StringValue` against reconstructed `FloatValue` hits the type-mismatch arm and emits on **every** report. **Reconstruct BOTH sides** (DP-H). This is the #1 trap and the reason AMD-51-INV-05 exists.
2. **Determinism vs schema read (AMD-50-INV-03).** Resolved by DP-K: the resolver is an immutable `Map<String,AttributeSchema>` snapshot built from `StandardCapabilities.all()` at the composition root and injected — NOT a live registry read inside `evaluate()` (the `QuantityValue.CATALOGUE` posture). Do NOT read a mutable registry from the rule and do NOT change `DerivationContext`; if you think you must, STOP and report (per the DP-K confirm-step).
3. **`Inf − Inf = NaN`.** Handle same-sign infinity equality *before* the `|a−b|` arithmetic, or the formula sees `NaN` (coincidentally returns the right answer for the wrong reason — make it explicit). Canonicalize `−0.0`→`+0.0` before compare.
4. **Both paths.** Reconstruction+compare must hold on **both** `onEvent` (~L434) and `processBatch` (~L572) (AMD-47-INV-02 / the M4.0a D-1 both-paths lesson). Keeping reconstruction inside `rule.evaluate` covers both by construction — verify.
5. **No concrete upcaster exists.** The `AttributeValueUpcaster` SPI has no production impl. Your reconstruction is a **separate** schema-driven step (AMD-51 §2.6) — do NOT extend the SPI signature, and do NOT block on a missing concrete upcaster. If a stored-value migration genuinely needs the SPI here, STOP and report (it would cross into AMD-52/migration territory).
6. **`shouldPublishDerived` is a string dedup** (~L758). It only *suppresses* on string equality, so it can't manufacture a change — but confirm it never suppresses a genuine typed-"changed" emit (§5 #10). Do not delete it.
7. **Preserve the String payload** (DP-G). The temptation is to put typed values into `StateChangedEvent` — that's AMD-52, explicitly OUT. Use the existing `priorValue`/`serializeAttribute` stringification for `oldValue`.
8. **Don't touch `CheckpointSerializer`, the event-store shape, or `module-info`.** If you think you must, STOP and report.
9. **`projectionVersion` bump is load-bearing.** Forgetting the 2→3 bump means no reconciliation runs and historical state silently keeps version-2 values. Bump the `HomeSynapseCore` literal and confirm the AMD-50 reconciliation path triggers (mismatch → backfill).
10. **Epsilon default is locked at `1e-9`** — do not "tune" it. The §5 #5b conversion-noise tests are the verification.
