## What This Implements

AMD-44 Stage 2 — the UX-role classification axis for entities. A kitchen smart plug exposes seven entities: the outlet (what the user cares about), voltage/RSSI/LQI sensors (device health), and a power-on-behavior switch (configuration). HomeSynapse currently treats all seven identically. Stage 2 adds the three-value `EntityRole` enum (PRIMARY / DIAGNOSTIC / CONFIG — Home Assistant's `EntityCategory` precedent, SYSTEM deliberately excluded), a **compile-time legality matrix on `EntityType`** (each entity type declares which roles it permits — adding a new EntityType forces the role declaration), and the `entityRole` field on `ProposedEntity` (adapter-proposed, defaults PRIMARY) and `Entity` (never null). Completing this **closes Workstream B** and unblocks Workstream C (the integration-api freeze waits on the post-B device model).

The `entity_profile_changed` `oldRole`/`newRole` extension is **taxonomy/JSON-schema-level only** (AMD-44 §4.3) — there is no typed Java event record for it (verified: only the `ENTITY_PROFILE_CHANGED` string constant at `EventTypes.java:107`). No event-model code changes in this WU.

## Settled Decisions (NOT open questions — implement as stated)

- **DP-1 — Matrix enforcement locus = record construction.** `Entity` and `ProposedEntity` compact constructors reject an illegal `(entityType, entityRole)` pair with `IllegalArgumentException`. AMD-44 §2.5.1 places enforcement "at adoption time and at reclassification time"; since every adoption and every reclassification constructs one of these records, the structural guard is strictly stronger — an illegal combination is unrepresentable. Precedent: the AMD-47-INV-04 `AttributeSchema` compact-ctor guard (accepted M4.B3, "stronger than the literal spec"). The future `DiscoveryPipeline`/`EntityRegistry` implementations inherit this for free and must not weaken it.
- **DP-2 — Null `entityRole` coerces to PRIMARY (both records).** AMD-44: ProposedEntity "default PRIMARY if unset by adapter"; Entity "never null; default PRIMARY". The compact constructors replace `null` with `EntityRole.PRIMARY` *before* the DP-1 guard runs. Every existing combination in tests/fixtures is therefore (type, PRIMARY), and every EntityType permits PRIMARY — zero existing-test semantic breakage.
- **DP-3 — Back-compatible convenience constructors.** `Entity` gains an 11-arg constructor (the old canonical signature) delegating with `EntityRole.PRIMARY`; `ProposedEntity` gains a 3-arg constructor doing the same. Existing callers (`EntityTest`, `TestEntityFactory`, `StubIntegrationContext`) keep compiling. Do NOT update `StubIntegrationContext` unless compilation forces it.
- **DP-4 — Defensive copies fold in.** While adding the `Entity` compact constructor, defensively copy both collection components (`List.copyOf(labels)`, `List.copyOf(capabilities)`); same for `ProposedEntity.proposedCapabilities`. The Javadoc already claims "unmodifiable" — this makes it true, mirroring the accepted M4.B-S1 `Device`/`ProposedDevice` pattern. Null collections now NPE (G8 confirms no caller passes null). **Do not** add null-guards for any other component (out of scope — keeps the diff minimal).
- **DP-5 — Preserve the existing `EntityType` declaration order** (LIGHT, SWITCH, PLUG, SENSOR, BINARY_SENSOR, ENERGY_METER — source order at `e73e199`, asserted by `EntityTypeTest.allExpectedValuesPresent`). The AMD §2.5.1 code block lists a different order; the **matrix content** governs, not the AMD's ordering. Do not reorder constants.
- **DP-6 — `legalRoles()` public accessor.** Beyond `allows(EntityRole)`, expose `public Set<EntityRole> legalRoles()` returning the immutable set (W23 plan names this surface; useful for UI/docs/adapter tooling). Store the field via `Set.copyOf(...)` so mutation attempts throw `UnsupportedOperationException`.
- **DP-7 — TypedIdTest FloorId fold-in is bundled here** (W23 carry-item; closes the B-S1 `[INFO]` follow-up). Test-only change in platform-api.

## Technical Specification

### `EntityRole` (enum) — CREATE
**Package:** `com.homesynapse.device`
**Values (this order):** `PRIMARY`, `DIAGNOSTIC`, `CONFIG`
Javadoc per AMD-44 §2.5 verbatim semantics:
- `PRIMARY` — "The entity's value is what the user installed the device to observe or control."
- `DIAGNOSTIC` — "The entity reports on the device's own health or infrastructure status."
- `CONFIG` — "The entity controls a device configuration parameter (e.g., polling interval, power-on behavior)."

Class-level Javadoc must state: three values only — no SYSTEM (HA scrubbed it pre-2021.11; visibility semantics live on `Entity.enabled` / future `visibleByDefault`); the Decision-9 coordinator convention (coordinator-state entities default DIAGNOSTIC); reclassification flows through `entity_profile_changed` with EntityId preserved (INV-CS-02). Cite Doc 02 §3.10 + AMD-44 §2.5. `@since 1.0`, `@see EntityType`, `@see Entity`.

### `EntityType` (enum) — MODIFY
Each constant declares its legal roles; the file's existing per-constant Javadoc is preserved and extended with a "Legal roles:" line.

```java
LIGHT(EnumSet.of(EntityRole.PRIMARY, EntityRole.DIAGNOSTIC)),          // status-indicator LEDs are real (Decision 3)
SWITCH(EnumSet.of(EntityRole.PRIMARY, EntityRole.DIAGNOSTIC, EntityRole.CONFIG)),
PLUG(EnumSet.of(EntityRole.PRIMARY)),
SENSOR(EnumSet.of(EntityRole.PRIMARY, EntityRole.DIAGNOSTIC)),
BINARY_SENSOR(EnumSet.of(EntityRole.PRIMARY, EntityRole.DIAGNOSTIC)),
ENERGY_METER(EnumSet.of(EntityRole.PRIMARY, EntityRole.DIAGNOSTIC));
```

```java
private final Set<EntityRole> legalRoles;

EntityType(Set<EntityRole> legalRoles) {
    this.legalRoles = Set.copyOf(legalRoles);   // immutable snapshot
}

/**
 * Whether {@code role} is a legal classification for entities of this type.
 * Enforcement is hard — adoption and reclassification reject illegal pairs;
 * there is no soft-fail or override (AMD-44 §2.5.1).
 *
 * @throws NullPointerException if role is null
 */
public boolean allows(EntityRole role) {
    Objects.requireNonNull(role, "role must not be null");
    return legalRoles.contains(role);
}

/** The immutable set of roles legal for this entity type. */
public Set<EntityRole> legalRoles() { return legalRoles; }
```

The full matrix (AMD-44 §2.5.1 — content authoritative, order per DP-5):

| EntityType | PRIMARY | DIAGNOSTIC | CONFIG |
|---|---|---|---|
| LIGHT | yes | yes | no |
| SWITCH | yes | yes | yes |
| PLUG | yes | no | no |
| SENSOR | yes | yes | no |
| BINARY_SENSOR | yes | yes | no |
| ENERGY_METER | yes | yes | no |

Every type permits PRIMARY (load-bearing for DP-2's zero-breakage claim).

### `Entity` (record) — MODIFY
New canonical signature (12 components — `entityRole` between `capabilities` and `createdAt`, per AMD-44 §2.5.3):

```java
public record Entity(
        EntityId entityId,
        String entitySlug,
        EntityType entityType,
        String displayName,
        DeviceId deviceId,          // nullable — helper entities
        int endpointIndex,
        AreaId areaId,              // nullable — inherits from device
        boolean enabled,
        List<String> labels,
        List<CapabilityInstance> capabilities,
        EntityRole entityRole,      // never null after construction; null coerces to PRIMARY
        Instant createdAt
) {
    public Entity {
        if (entityRole == null) { entityRole = EntityRole.PRIMARY; }
        if (!entityType.allows(entityRole)) {
            throw new IllegalArgumentException(
                "entityRole " + entityRole + " is not legal for entityType " + entityType);
        }
        labels = List.copyOf(labels);
        capabilities = List.copyOf(capabilities);
    }

    /** Convenience constructor preserving the pre-AMD-44 11-arg signature; entityRole defaults to PRIMARY. */
    public Entity(EntityId entityId, String entitySlug, EntityType entityType, String displayName,
                  DeviceId deviceId, int endpointIndex, AreaId areaId, boolean enabled,
                  List<String> labels, List<CapabilityInstance> capabilities, Instant createdAt) {
        this(entityId, entitySlug, entityType, displayName, deviceId, endpointIndex, areaId,
             enabled, labels, capabilities, EntityRole.PRIMARY, createdAt);
    }
}
```

Note the guard ordering: coercion → matrix guard → copies. `entityType` null was never guarded before — do NOT add a guard (DP-4 boundary); the matrix guard will NPE on a null `entityType` via the `allows` call, which is acceptable and must be documented in a code comment, not "fixed". Javadoc: add `@param entityRole the UX-role classification, never {@code null} after construction ({@code null} coerces to {@link EntityRole#PRIMARY}); must satisfy {@code entityType.allows(entityRole)}` and update the class Javadoc to mention the role axis + the reclassification path (`entity_profile_changed`, EntityId stable per INV-CS-02).

public record ProposedEntity(
        int endpointIndex,
