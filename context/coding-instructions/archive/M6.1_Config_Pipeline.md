## What This Implements

M6.1 is the **non-crypto config pipeline** — the first of M6's four first-class pieces. It turns the on-disk YAML configuration into the validated in-memory `ConfigModel` and serves it to subsystems. Concretely: parse YAML 1.2 (safe-by-default), compose + validate against JSON Schema, build the `ConfigModel`, and expose it through `ConfigurationService` (load/read) and the integration-scoped `ConfigurationAccess`. It also lands the M6-config-block contracts that belong to the load path: the `(major,minor)` schema model (AMD-67), the hybrid directory layout + path-traversal guard (AMD-71), the `ConfigurationChangeListener` interface + registration (AMD-66 — *defined* here, *exercised under swap* in M6.4), and the `config.validation_completed` observability event (AMD-70 — the load/validate half). The **reload** path (atomic swap, `config.section_reloaded`, listener invocation) is **M6.4**.

## Settled Decisions (do not re-open)

- **DP-1 (schema version shape):** `(configSchemaMajor, configSchemaMinor)` per AMD-67. The on-disk YAML top-level key is `schema_version: { major: N, minor: M }` (object form — explicit, no string-parse ambiguity). Migrator triggers on **major** mismatch only (AMD-67-INV-02).
- **DP-2 (validation error model):** allErrors mode (collect all `ConfigIssue`s in one pass — the frozen `ConfigValidator` contract). On `load()`: `Severity.ERROR` → revert that key to its schema default (degraded-but-functional start); `Severity.FATAL` → `ConfigurationLoadException` (Doc 06 §3.6, source-verified `ConfigurationService` Javadoc).
- **DP-3 (config-dir resolution — the `[AMD-71-A]` resolution):** `StandardConfigurationService` takes the **resolved config-dir `Path` as a constructor parameter**, supplied by the composition root (`com.homesynapse.app`, which already `requires` both platform + config) from `PlatformPaths.configDir()`. **M6.1 adds NO `requires com.homesynapse.platform`** — this keeps config's module-info unchanged and preserves the no-new-edge property the M6.2 E2 bridge depends on. (Alternative `config→platform` edge is REJECTED.)
- **DP-4 (listener registration):** constructor-injected `Map<String, ConfigurationChangeListener>` keyed by `sectionPath()`, no `ServiceLoader` (DEC-M3-16); duplicate section path → construction-time `IllegalArgumentException` (AMD-66 §2.4). _[CORRECTED by the RULING ADDENDUM: `List<ConfigurationChangeListener>` ctor; the service builds the map.]_
- **DP-5 (Clock):** the load path stamps `ConfigModel.loadedAt`/`fileModifiedAt` from an **injected `Clock`** (NOT `Instant.now()`) — required for the §4c arch rule and deterministic tests. _[D3 doc-accuracy finding (2026-06-10): as written this contradicts the frozen `ConfigModel` Javadoc + Doc 06 §3.5 — `fileModifiedAt` is the optimistic-concurrency token and must be the REAL file mtime when the file exists; the Clock stamps only `loadedAt` (and `fileModifiedAt` solely in the absent-file zero-config case). Implemented correctly; prose corrected here at archival so a future reader doesn't "fix" the implementation backward.]_

## What to Watch Out For

- **`Clock` injection (§4c — `com.homesynapse.config` is NOT whitelisted).** Do NOT use `Clock.systemUTC()`, `Instant.now()`, `System.nanoTime()`, or `System.currentTimeMillis()` in this module's main OR test code — the `NO_DIRECT_TIME_ACCESS` ArchUnit rule scans non-whitelisted classes and fails `./gradlew check`. Inject `Clock.fixed(Instant.parse("2026-01-01T00:00:00Z"), ZoneOffset.UTC)` via constructor/`@BeforeEach`. (M2.4/M2.5 tripped this; `coder-lessons.md` 2026-04-10.)
- **`ConfigurationValidationException` is in `com.homesynapse.event`, NOT config (MODULE_CONTEXT gotcha).** Do not duplicate it; reference via import. `ConfigurationLoadException`/`ConfigurationReloadException` ARE in config.
- **`ConfigChange` vs `MigrationChange` are different types** (the Doc-06 name collision; resolved by rename). Don't confuse the reload-diff record with the migration record.
- **`SchemaRegistry`/`ConfigValidator` take JSON-text `String`s, never library types** (JPMS hygiene — the networknt `JsonSchema` type must never appear in the public API). Parse JSON strings internally.
- **`requires transitive com.homesynapse.event` is correct, not a bug** (event-model types — `HomeSynapseException`, `ConfigurationValidationException` — leak through the public API; the `requires transitive`↔`api` lockstep is already satisfied by `api(project(":core:event-model"))` in `build.gradle.kts`). Do not "fix" it to plain `requires`.
- **Path-traversal guard must be canonicalization-based** (`Path.toRealPath()` containment under `configDir()/integrations/`), NOT string-prefix matching — symlink-safe (AMD-71 §2.3). This is the security-load-bearing detail.
- **`!include` is one level deep** — invoke the included-file parser **without** the `!include` tag handler registered, so a nested include is an unknown-tag FATAL error (AMD-71 §2.2).
- **Derived/observability event time:** `config.validation_completed` uses null or inherited `eventTime`, never `Instant.now()` (the standing event-time rule). `ingestTime` is the publisher's concern.
- **Do NOT touch `SecretStore`, the key manager, or any at-rest/crypto path** — that is M6.2/M6.3. M6.1 is non-crypto.
