## 5. M4.0b scope and the gating questions to resolve

### 5.A Scope (from PLAN-M4-CONSOLIDATED-v2 + the coder-handoff M4.0b description ‚Äî confirm against source)

1. **Replace `MinimalProjectionAdvancer` with `DispatchingProjectionAdvancer`** (Research 8 REC-28): constructor-injected, package-private per-event-type handlers, **no `ServiceLoader`** (DECIDE-04).
2. **Promote a real production `DerivationRule`**: lift the change-detection logic out of the testFixtures `EchoStateRule`; extend it from string-only to typed `AttributeValue` (including the Research-8 `QuantityValue` / `ArrayValue`). It **must publish `state_changed`**, or state/numeric triggers never wake.
3. **REPLAY semantics:** on REPLAY the rule **re-derives but does not re-publish** (publish is LIVE-only ‚Äî AMD-41 ¬ß3.2.2).
4. **Version bump ‚Üí reconciliation:** shipping a real rule bumps `projectionVersion` 1 ‚Üí 2, which triggers reconciliation / replay-from-zero on first boot (AMD-41 ¬ß3.2.4) and records `reconciledToVersion = 2`.
5. **One-shot backfill:** apply re-derived drafts to in-memory state during the 1 ‚Üí 2 replay **only**, gated to that version boundary. Applying them on later replays would double-increment `stateVersion` (the idempotency cursor). This is a **locked** decision.
