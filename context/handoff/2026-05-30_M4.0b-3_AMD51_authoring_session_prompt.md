**The four ratified strategic calls (Nick, 2026-05-30):**
1. **Defer the deadband.** Structural-equality + epsilon is the M4.0b-3 scope. Future home for an optional deadband = `AttributeSchema` (analog types only). Do not implement now.
2. **Float epsilon — pinned, no latitude.** AMD-51 contract is the total form `changed ⟺ |a−b| > max(absEps, relEps·max(|a|,|b|))` with explicit IEEE-754 totality: NaN↔number = changed, NaN↔NaN = unchanged, `−0.0` == `+0.0`, ±Inf defined. A pure relative epsilon is rejected (explodes near zero). Defaults `absEps = relEps = 1e-9` are a *correctness* epsilon (FP-noise), **not** a perceptual deadband — verify magnitude against real M4 sensor data before locking.
3. **Hand-roll units — already satisfied upstream.** `QuantityValue` canonicalizes at construction (AMD-47), so both operands reach the comparator already in the same unit → **the comparator does zero unit work.** No JSR-385 / Indriya, no LTD-10. AMD-51 must **cite AMD-47's canonicalize-at-construction**, not re-specify REC-93 as new work (this resolves the REC-93 double-booking).
4. **Stage AMD-51 before AMD-52.** AMD-51 ships the comparator with the **String `StateChangedEvent` payload preserved**. AMD-52 (typed payload) waits behind its own serializer/replay design beat — tracked as **OQ-05-08** (the `CheckpointSerializer`/Jackson + event-store + replay-determinism blast radius, design-track-map §3). Do **not** touch the event payload in AMD-51.

**NQ-10-5 — comparator contract shape (ratified placement):** an **external `AttributeValueComparator` in `core/state-store`** (`com.homesynapse.state`), carrying a `ComparisonPolicy` (epsilon now; future deadband). **Not** a polymorphic method on the `AttributeValue` sealed interface — that would drag projection policy into the device-model data layer. Two binding contract notes:
