## What AB-1 + AB-2 + AB-4 is (scope — from the charter §2 + the A1–A4 rulings)

**The Seam-1 insight (charter §1, sharpened by the Track-3 red-team):** five things go live or lock at the *same instant* `main()` exposes the system — the HTTP surface becomes live + authenticated (AB-1), the cipher activates so a read-path decrypt failure is a live availability risk (AB-2/C2), and the one-way crypto doors lock as the first encrypted row is written (AB-4/F1/F2/F3). They land together for go-live coherence and because you must not expose-and-write before the cipher is on.

The instruction must cover three first-class pieces:

1. **AB-1 — Auth + bind posture (C1 / INV-SE-02).** Build `AuthMiddleware` (zero-impl today) enforced on **every** external route — including `/internal/*`, which today sits outside even the readiness gate — at the router **before any path resolves** (canonicalize paths first — R-δ AX-1, the HA CVE-2023-27482 traversal class). `RateLimiter` (zero-impl today). **Bind posture = loopback-default (A1); LAN exposure is explicit, authenticated opt-in; never bind-all** (the CVE-2026-34205 CVSS-9.7 class). **Auth model = token issuance (A2):** bearer tokens minted at first-run/pairing; the **WebSocket is an external interface and is authenticated**; zero-config stays **authenticated** — no default/shared bootstrap secret (Mirai class), no pre-auth account enumeration (HA 2023.12 class), no unauthenticated stream/media path. **Carry the enterprise per-scope/per-site-claims hook** (A2 caveat — design the token model to anticipate scoped claims even if MVP issues them simply). **Wire into AB-3's `exposeHttpSurface()` seam:** AB-1 adds auth + makes the production phase progression invoke it behind auth + the loopback bind (AB-3 left it test-only / closed). **Fold CC-1 as a confirm-gate:** assert the bound socket answers only on loopback unless LAN opt-in is configured.

2. **AB-2 — Read contract, fail-closed half (C2 / F-A1).** On GCM-auth-fail / missing-or-corrupt root key / chain-hash failure → **fail the read batch closed with a distinct, loud error** (A3; R-α REC-234 — a flat catch-and-degrade masks CASE-b tampering, disproven). Today `SqliteEventStore.fromRow`/`decryptStoredPayload` **throws and aborts the whole replay** (OR-RF-DECRYPT) — wire the fail-closed contract. **Design the degrade seam now** (the cause-carrying `DegradedEvent` type — confirm the existing `core/event-model/.../DegradedEvent.java`; the `(scope, key_version)`-keyed cause lookup; the chain-validity check) but the **degrade behavior is OUT** (crypto-shred WU). Carry the **rotate-DEK-on-restore boot invariant** (R-α REC-235 / AMD-94). **F4 pin:** the degrade half + the chain-validity check stay DISABLED until `chain_hash` + mandatory startup-verify are live (32-byte ZERO today). **Add tests for both decrypt-failure paths** (missing-cipher-on-read, shredded/undecryptable-key-on-read — currently none).
