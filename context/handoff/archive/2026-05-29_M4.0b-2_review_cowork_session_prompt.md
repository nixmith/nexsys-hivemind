## 4. What M4.0b-2 was (the WU under review)

**Instruction (the contract you review against):** `context/coding-instructions/M4.0b-2_Version_Transition_Backfill.md`.

M4.0b-2 implements the **ratified AMD-50** for the **1→2 transition** on M4.0b-1's existing **string** change-detect rule. It was deliberately scoped to the **full, general AMD-50 mechanism** (not a 1→2 shortcut) so that **M4.0b-3 is a clean typed rule-swap** that reuses this reconciliation path unchanged. Modules: **`core/state-store`** + a one-line **`lifecycle`** bump. Expect **no** `module-info.java`/`build.gradle.kts` changes.

The seven behavioral contracts:
1. **`projectionVersion` 1→2** at `HomeSynapseCore` (the reconciliation trigger).
2. **`backfillActive` provenance gate** — set `true` in `StateProjection.initialize()`'s reconciliation branch (version mismatch, escape hatch off); cleared at `onCaughtUp()` (REPLAY→LIVE).
3. **Backfill apply of re-derived drafts during REPLAY** via a **narrow attribute-only write** — gated on **BOTH `onEvent` and `processBatch`** (the D-1 lesson). No publish; no second `stateVersion` increment.
4. **Supersession suppression** in `applyToState`'s `state_changed` branch — under the gate, a logged prior-version `state_changed` advances `stateVersion` (cursor) but does **not** write `attributes`/`lastChanged`.
5. **Remove `Clock` from `DerivationContext`** (`(priorState, envelope, clock)` → `(priorState, envelope)`); update both `StateProjection` construction sites + the rule + tests; the projection keeps its own clock.
6. **AMD-50-INV-04** — confirm the only replay-from-zero path is the reconciliation branch.
7. **Six AMD-50 §5 tests** (rebuild-idempotency; backfill≡native on attribute *values*; one-shot; steady-state safety; **supersession/generality**; determinism).

**The timestamp fix (important review target):** the backfill's `lastChanged` must come from the **causing event's time** (event-time, inherited via the derived draft's `eventTime`), **not** the projection wall-clock — for semantic correctness and rebuild determinism. After this WU, `lastChanged` has a **deliberately documented interim mixed semantics**: event-time when set by the backfill, wall-clock when set by the LIVE `applyToState`. That is a **conscious interim, not a bug** (the new code is correct; the pre-existing wall-clock wart is untouched). The full timestamp-model unification is a **separate, scheduled follow-up WU** (needs a design beat; possibly amendment-worthy; read-path blast radius). Do not treat the interim as a defect, and do not let the closeout absorb the follow-up.
