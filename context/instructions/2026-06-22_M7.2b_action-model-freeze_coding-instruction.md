## What This Implements

M7.2b **freezes the action model** now that the run/execution engine acts (M7.2a-1/2 GREEN). Three things, in priority order:

1. **The run-coupled-reliability terminal contract (Doc 16 §3.4 / C-SA-5).** The terminal states (`RunStatus`: COMPLETED/FAILED/ABORTED/INTERRUPTED/CONDITION_NOT_MET) and the `automation_completed` `failureReason`/`abortReason` fields **already shipped** in M7.2a-1/2. This work **binds them into a tested contract**: *every* Run reaches a deterministic terminal state with a recorded, machine-readable reason; **no autonomous retry**; and it adds the **run-coupled fail-closed-read coupling** — a Run whose trigger-time snapshot read fails closed terminates `FAILED` with a `failureReason` identifying the degraded read (the run-coupled half of app-bootstrap A3). This half is mostly **tests + a small fail-closed handler**, not new engine structure.
2. **A minimal computed-param resolution seam at run-init (Doc 16 §3.2 beat a).** A `ComputedValue` sealed type (small bounded operator set) resolved against the **captured trigger-time snapshot** (AMD-03 single snapshot) in the run-init path, **before action execution**, so an action sees a concrete typed value. **Minimal / present-not-built-out (Nick, 2026-06-22):** the seam + a small operator set, exercised programmatically in tests; **YAML-loading of computed values and the full component model are OUT** (deferred). The `ActionExecutor` seam is **frozen** — resolution happens *before* it and does not change its signature.
3. **Freeze no-engine-retry (D2/REC-162 / AMD-90-INV-01).** Confirm + test that the action layer never autonomously re-dispatches; record the disposition (retry is post-MVP, opt-in, device-class-gated — not in V1, not in the engine).

## Technical Specification

### 1. `ComputedValue` (sealed interface) + minimal permits — `com.homesynapse.automation`
**Responsibility:** a bounded, total, side-effect-free typed derivation over {trigger-time snapshot attributes, time} that resolves to a concrete `AttributeValue` at run initiation (Doc 16 §3.2/§4; C-SA-2). **Not** a `ConditionDefinition` permit; **not** an expression string.

```java
/** Total, side-effect-free; resolves to a concrete AttributeValue against the run-init context. */
public sealed interface ComputedValue permits LiteralValue, AttributeRef, AggregateValue {
    AttributeValue resolve(ComputedValueContext ctx);   // total; never I/O; never throws on valid typed inputs
}
```
- **`ComputedValueContext`** (record): `StateSnapshot snapshot`, `Instant resolutionTime` (the run-init time, from the injected `Clock` — never `Instant.now()`). (Component parameters are a deferred input — omit for V1; do not add a parameter map yet.)
- **Permits (the minimal set — exactly these three):**
  - **`LiteralValue(AttributeValue value)`** — resolves to its constant.
  - **`AttributeRef(EntityId entity, String attribute)`** — resolves to `ctx.snapshot().states().get(entity).attributes().get(attribute)`; **absent → a defined typed-absent** (return a documented sentinel/`Optional`-style absent, OR fail-closed per §3.4 — see Error Handling; pick one and pin it in a test, do not leave it implicit).
  - **`AggregateValue(Selector selector, String attribute, AggregateOp op)`** — a **bounded fold** (`op` ∈ {`SUM`,`AVG`,`MIN`,`MAX`,`COUNT`}) over the **resolved** entity set's numeric attribute values from the snapshot. **No recursion** (operands are not `ComputedValue`s); bounded by the resolved `Selector`'s finite entity set. Non-numeric/absent members: defined skip-or-fail rule, pinned in a test.
- **Static analyzability (the superiority point):** because every permit is total over a typed finite input set, resolution is deterministic and replay-safe (INV-TO-02). No I/O capability on the type (C-SA-2).

### 2. Run-init computed-param resolution (into the existing `initiateRun` path)
In `StandardRunManager`'s run-init flow, **after** the trigger-time snapshot is captured and **before** the actions reach `ActionExecutor.execute(...)`, resolve any `ComputedValue` occupying an action value position into a concrete value, producing a **resolved action list** the executor consumes. The executor seam is unchanged.
- **Scope (minimal):** resolve `ComputedValue`s in **`CommandAction.parameters()`** value positions only (a `Map<String,Object>` value may be a `ComputedValue` → replace with the resolved `AttributeValue`/concrete value). Other action types: out of scope for V1 (leave their literal params untouched).
- **Single snapshot (AMD-03):** resolution uses the **same** captured trigger-time snapshot the condition gate used — not a fresh read. Determinism + replay depend on this.
- **A resolved value is concrete:** the executor never sees a `ComputedValue` (so `ActionExecutor`/`CommandDispatchService`/`CommandValidator` are unchanged and never learn about `ComputedValue`).
- **Loading is OUT:** the YAML loader does **not** parse `ComputedValue` in V1 (component-model territory). For V1 the seam is exercised by **programmatically-constructed** `ComputedValue`s in tests. State this in the MODULE_CONTEXT note.

### 3. Terminal-contract binding + the fail-closed-read coupling (Doc 16 §3.4 / C-SA-5)
- **Bind (mostly tests):** assert every Run path reaches a deterministic terminal `RunStatus` with the appropriate recorded reason and exactly one `automation_completed`, and that **no path re-dispatches** (no retry). The FSM already performs the transitions (M7.2a-1/2) — this is the contract + its test coverage, not new transition logic.
- **Add — run-coupled fail-closed read:** if the trigger-time snapshot read **fails closed** (the read path can throw post-AB-2, e.g. a typed decrypt/fail-closed exception), the Run terminates **`FAILED`** with a `failureReason` identifying the degraded read, and the failure is explainable (an `automation_completed(FAILED, failureReason=...)`). It does **not** proceed on partial/ambiguous state. (The later `DegradedEvent` degrade *behavior* is gated on A3's F4 — OUT here; do **not** build it.)

### Event Types Produced or Consumed
- **Produces:** none new. Uses the existing `automation_completed` (terminal state + reason). **Counts unchanged: 71/41/53.**
- **Consumes:** the trigger-time `StateSnapshot` (state-store) for resolution + condition eval.

### Error Handling
| Condition | Behavior |
|---|---|
| `AttributeRef`/`AggregateValue` member absent or non-numeric | A **defined, total** rule (typed-absent sentinel or skip) — never throw on valid typed inputs (C-SA-2); pin the chosen rule in a test |
| Trigger-time snapshot read fails closed | Run terminates `FAILED` with a `failureReason` identifying the degraded read (§3.4); explainable; no retry |
| (Never) autonomous retry | Not implemented — AMD-90-INV-01 frozen |

Error messages: Register C voice (direct, neutral, no self-reference/apology).

## What to Watch Out For
- **RECORD COMPONENT / STATIC-FACTORY COLLISION (the M7.2a-2 gate-fix lesson — do not repeat):** a record component named `X` already defines the public accessor `X()`; **never** declare a `static` factory (or any method) named `X()` in that record. Before handoff, scan every new record (e.g. `ComputedValueContext`, any value record) — for each component name, confirm no same-named method. (Factories with non-component names are fine.)
- **The `ActionExecutor` seam is FROZEN** — resolve *before* it; do not change `execute(...)` or touch `CommandDispatchService`/`CommandValidator`.
- **Single snapshot (AMD-03):** resolve against the *captured* trigger-time snapshot, not a fresh `getSnapshot()` — a second read breaks determinism/replay.
- **`ComputedValue` must be total + side-effect-free** — no I/O, no recursion, bounded folds only. A computed value that can throw or read I/O reintroduces the silent-failure class the sealed model forecloses (SP1).
- **Residency (AMD-92-INV-01):** `ComputedValue` must never enter an event payload.
- **VT pinning (LTD-11):** `ReentrantLock`, never `synchronized`, in any touched concurrent path.
- **`Clock` injection (§4c):** non-whitelisted module — no `Instant.now()`/`System.nanoTime()` in production or test; use `Clock.fixed(Instant.parse("2026-01-01T00:00:00Z"), ZoneOffset.UTC)`.
