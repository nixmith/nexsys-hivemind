## Settled Decisions (deliberated in the M9-authoring lane — NOT open questions)

- **DP-1 (Ownership + naming):** The routing subscriber lives in `integration-runtime` inside the supervisor's orbit (its MODULE_CONTEXT Phase-3 note specifies exactly this). `SUBSCRIBER_ID = "integration_supervisor"` — one service, one snake_case id, the `automation_engine` precedent.
- **DP-2 (Envelope construction = causation join):** `command_dispatched` carries no command name/parameters (G3). The subscriber's filter covers BOTH `command_issued` and `command_dispatched` (one `SubscriptionFilter`, `SubjectType.ENTITY`, `EventPriority.DIAGNOSTIC` minimum — `command_dispatched` is published DIAGNOSTIC). `command_issued` envelopes populate a bounded join-cache keyed by `eventId().value()` (Ulid); on a LIVE `command_dispatched`, join via `event.causalContext().causationId()` → build `CommandEnvelope(EntityId.of(dispatched.targetEntityRef()), issued.commandType(), decodedParams, causationId, event.causalContext().correlationId(), IntegrationId.of(dispatched.integrationId()))`. Join miss ⇒ structured WARN (`integration.route_join_miss`) + skip — no dispatch, no result event (the ledger's timeout owns the outcome).
- **DP-3 (Parameter decoding is INJECTED):** `CommandIssuedEvent.parameters` is a serialized JSON String; integration-runtime has NO JSON library and gains no Jackson edge. The assembly takes a `Function<String, Map<String, Object>> parameterDecoder` injected from the composition root (lifecycle already has Jackson transitively via persistence — construct the decoder there exactly the way M7.4a injected the serializer in the opposite direction; empty/`"{}"`/null-safe → `Map.of()`; decode failure ⇒ WARN + `Map.of()`, never an exception on the bus thread).
- **DP-4 (Cache bounds):** Join-cache capacity 1024, insertion-ordered drop-oldest with a structured WARN (`integration.command_cache_evicted`), entry evicted on successful join. Populated in ALL modes (a `command_issued` delivered during TRANSITION may legitimately join a LIVE `command_dispatched` across the flip); **dispatch fires only for LIVE `command_dispatched` events** (`replayMode` guard identical in shape to `StandardCommandDispatchService`). Clock-injected timestamps only (no TTL sweep in M9.1 — the ledger owns timeout semantics; the cap owns memory).
- **DP-5 (Handler invocation + failure boundary):** `handle(...)` is invoked on the owning adapter's **single-threaded per-adapter command executor** (virtual thread, named `integration-cmd-<type>`, FIFO per adapter — Doc 08 §3.10 step 1) — never on the bus subscriber thread. Post-join failure taxonomy, each publishing `command_result` (payload `CommandResultEvent(targetEntityRef, commandType, outcome, failureReason)`, priority **CRITICAL** on failure per Doc 01 §4.3 dual default, origin SYSTEM, causation chained from the `command_dispatched` envelope): unknown/not-running integrationId → outcome `"integration_unavailable"`; `commandHandler() == null` (read-only adapter — legal per gotcha) → `"unsupported"`; `handle(...)` throws → `"handler_error"` (+ classify: the throw also feeds the health error window; `InterruptedException` → restore interrupt, treat as shutdown, no result event). **On a normal `handle(...)` return the router publishes NOTHING — the adapter owns the eventual `command_result` (Doc 08 §3.10 step 7). The router never fabricates success.**
- **DP-6 (Integration identity, M9.1-interim):** `IntegrationIds.deriveStable(type)` = `Ulid` from the first 128 bits of SHA-256 of `"homesynapse:integration:" + integrationType` — stable across restarts (a boot-random id would orphan every `Device.integrationId` on restart), deliberately NOT a time-ordered ULID (documented deviation from the LTD-04 `generate()` guidance; wrapper type preserved; **[Design point] DP-B in the lane return — Nick rules the durable identity story before M9.2's real device adoption makes it one-way**). The supervisor resolves each factory's id through this utility at registration.
- **DP-7 (Ctor + skip-if-empty):** New canonical 7-arg ctor `HomeSynapseCore(Path, Path, HomeSynapseConfig, Clock, HomeId, PayloadCipher, List<IntegrationFactory>)`; the 6-arg delegates with `List.of()` (5-arg chain unchanged) ⇒ zero caller edits (G8). `integrationFactories` null → NPE (`Objects.requireNonNull`); **empty → Phase 6 is skipped entirely** (no supervisor, no router, byte-identical runtime behavior to `52824e9` for `Main`) — the AB-3/AB-4 held-not-consumed precedent.
- **DP-8 (Phase-6 boot posture):** Construct assembly → `subscribeRuntime` the router (AFTER projection LIVE, same ordering invariant + comment discipline as 3.4b/3.4c; `coalesceExempt=true` — a coalesced `command_dispatched` is a command that never reaches the device) → `supervisor.start(factories)` bounded by `.get(30, SECONDS)`; timeout/exception ⇒ structured ERROR + **boot continues** (INV-RF-01: integrations never take down core; the health surface carries the failure). `recordSubsystem("integration", LifecyclePhase.INTEGRATIONS, …)` mirroring the existing per-phase recording.
- **DP-9 (Lifecycle events, zero mint):** Supervisor publishes the EXISTING types only — `integration_started` (adapter RUNNING), `integration_stopped` (clean stop, with reason), `integration_restarted` (TRANSIENT restart cycle), `integration_health_changed` (every state transition; CRITICAL priority on transitions to FAILED, NORMAL otherwise) — `publishRoot`, `SubjectRef.integration(id)`, `EventOrigin.SYSTEM`, payload records from integration-api (WU-M9.1 §A12). Counts stay 71/41/53.
- **DP-10 (Minimal health FSM — the M9.1 slice):** States exercised: HEALTHY ↔ restart cycle → FAILED. `initialize()`/`create()` throwing `PermanentIntegrationException` → FAILED immediately (no retry). `run()` exiting/throwing → classify: PERMANENT → FAILED; TRANSIENT → restart with descriptor `BackoffParameters` (exponential, capped), FAILED when restarts exceed `HealthParameters.maxRestarts` within `restartWindow`; SHUTDOWN_SIGNAL → clean stop. **DEGRADED/SUSPENDED, the probe ladder, suspension cycles, JFR/resource quotas, planned-restart suppression (Doc 05 §3.14), dependency-graph (Kahn) ordering, and integration-scoped context wrappers are OUT (forward pointer: the post-hero supervisor-breadth unit; NQ-6 validates restart defaults before it).** `IntegrationHealthRecord` snapshots populate the fields this slice tracks; untracked windows carry empty `SlidingWindow(size, 0, 0.0)` values; `lastKeepalive` stays null until reported (nullable by contract).
- **DP-11 (Thread allocation):** Descriptor-driven: `IoType.NETWORK` → virtual thread (`Thread.ofVirtual()`); `IoType.SERIAL` → dedicated platform thread (`Thread.ofPlatform()`), named `integration-<type>-0` — both branches implemented now (the Zigbee adapter arrives SERIAL at M9.4). `ReentrantLock` only; no `synchronized` anywhere (LTD-11 + arch rule).
- **DP-12 (Context composition, M9.1 slice):** Per-adapter `IntegrationContext(deriveStable(type), type, <shared EventPublisher>, <real EntityRegistry>, <real StateQueryService>, new SupervisorHealthReporter(...), <configAccess>, null, null, null, null, null)` — the 5 nullable services stay null (the fake declares no `RequiredService`; wiring real SCHEDULER/TELEMETRY arrives with the Zigbee descriptor at M9.4). **`configAccess` is wired per the CORRECTED Files-table row [HUB-AUDIT CORRECTION 2026-07-02]: the preferred path exposes the EXISTING `ScopedConfigurationAccess` (config module, package-private, per-integration-scoped) via a small public factory — the original "no implementation exists anywhere" premise was refuted at hub audit; `HomeSynapseCore` holds only `ConfigurationService`, so lifecycle composes the factory over `getCurrentModel()`. The compact ctor's null-check makes SOME wiring mandatory even though the fake never reads config.** Integration-SCOPED registry/query wrappers remain deferred with the breadth unit (config scoping now arrives via the preferred path); record the composition in MODULE_CONTEXT.

Behavioral contracts: null-checks all six; builds ONE `StandardIntegrationSupervisor` and ONE `CommandRoutingSubscriber` sharing supervisor state (the ledger's one-object-two-views precedent is acceptable but not required — two collaborating package-private objects are fine); the returned `Subscriber` is the ONLY exported view of the router. **Because `Components.subscriber` puts the event-bus `Subscriber` type on this module's exported API, the module-info/Gradle lockstep in §5 is mandatory (the 3×-recurring `-Xlint:exports` trap).**

### `StandardIntegrationSupervisor` (package-private final class, `implements IntegrationSupervisor`)
Implements all 9 interface methods. Core behavioral contracts:
- `start(factories)`: for each factory — `descriptor()` (pure; validate `integrationType` non-blank, duplicate types → `IllegalArgumentException` listing the duplicate), resolve id via `IntegrationIds.deriveStable`, `create(context)` (PermanentIntegrationException → record FAILED + `integration_health_changed` CRITICAL + continue with remaining factories), `initialize()` (same failure contract), then launch `run()` on the DP-11 thread. Returns a future completing when every factory has been LAUNCHED-or-FAILED (never blocks on adapter health). Idempotency: second `start(...)` while running → `IllegalStateException`.
- `stop()`: reverse-registration-order shutdown; per adapter: interrupt run-thread, `close()` bounded by a 10 s grace (Doc 05 §3.6 default; log abandonment on overrun, proceed), publish `integration_stopped`. Safe when never started (no-op) — `doTeardown` is null-guarded but boot-failure paths may reach it in odd states.
- `startIntegration(id)`: manual restart, only from FAILED → `IllegalStateException` otherwise (MODULE_CONTEXT gotcha, verbatim contract).
- `restartIntegration(id)`: stop-then-start of one adapter; sets `plannedRestart=true` on the health record for the duration (flag only in M9.1 — the §3.14 suppression behaviors are deferred breadth); publishes `integration_restarted`.
- `health`/`allHealth`/`isRunning`/`registeredIntegrations`: snapshot reads guarded by the state lock; `IntegrationHealthRecord` populated per DP-10.
- All time via the injected `Clock`. All mutation under `ReentrantLock`.

### `CommandRoutingSubscriber` (package-private final class, `implements Subscriber`)
- `setMode(SubscriberMode)`: `replayMode = mode != SubscriberMode.LIVE;` (verbatim pattern).
- `onEvent(envelope)`: `CommandIssuedEvent` payload → cache put (all modes, DP-4). `CommandDispatchedEvent` payload → if `replayMode` return; join per DP-2; resolve adapter via supervisor state; invoke per DP-5 on the per-adapter executor. Any other payload → return (filter should preclude).
- Never throws from `onEvent` (a throw is a subscriber crash — catch-all with structured ERROR; the bus's isolation is the backstop, not the plan).

### `ExceptionClassifier` (package-private, static `classify(Throwable)`)
`PermanentIntegrationException` → PERMANENT · `InterruptedException` → SHUTDOWN_SIGNAL · `OutOfMemoryError`/`LinkageError` → PERMANENT · everything else incl. unknown `RuntimeException` → TRANSIENT (Doc 05 §3.7 — the HA anti-pattern guard; MODULE_CONTEXT gotcha verbatim).

### `SupervisorHealthReporter` (package-private, `implements HealthReporter`)
`reportHeartbeat()` / `reportKeepalive(Instant)` / `reportError(Throwable)` / `reportHealthTransition(HealthState, String)` write through to the owning integration's supervisor state under the lock (heartbeat/keepalive timestamps from the injected `Clock`; errors increment the error window count). No evaluation logic in M9.1 (no heartbeat-timeout sweep — deferred breadth); the record surface is honest storage.

- Produces: `command_result` (failure outcomes only, DP-5), `integration_started`, `integration_stopped`, `integration_restarted`, `integration_health_changed` (DP-9). **All pre-existing. ZERO new event types — 71/41/53 pins UNCHANGED (G6).**

### Error Handling (router + supervisor surface)
| Condition | Mechanism | Recovery |
|---|---|---|
