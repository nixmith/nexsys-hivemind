**ALL THREE variants are DEFINED now; matching is IMPLEMENTED for the first two only (DP-2 verbatim: "define all three variants now … and implement matching for only the first two … Wave-2's first TS0601 then populates a variant that already exists — no sealed-hierarchy change, which is the expensive kind").**

### 1.3 The namespace convention on `profileId` (dossier §C + Doc 18 §3.5(a)/(b), Locked — convention now, no schema change)

## Technical Specification (contract level — implementation freedom inside the contracts)

**Interview FSM (Doc 08 §3.4, verbatim-constrained):** announce → node-desc → active-EPs → simple-desc → Basic-cluster reads → profile-match → `device_discovered` → adoption → bind → configure-reporting. Per-step timeout 10 s; whole-interview 60 s; 3 retries at 5/15/30 s backoff (Clock-injected — never wall-clock). PARTIAL flag on step failures per §3.4 (adoptability is the Device Model's call downstream). **Sleepy queue:** pending interviews resumed when the device sends ANY frame; 24 h expiry with a structured log entry. **Measured field truths (charter, binding):** read Basic from the FIRST APPLICATION endpoint (the Hue light lives on EP 11, not EP 1 — measured); Green Power EP 242 + unknown manufacturer clusters (0xFC01/0xFC04/0xFC57 measured) are SKIPPED GRACEFULLY, never interview failures.

**Ingestion:** drain via `drainPendingCallbacks()` (§G: BOUND the callback queue and DRAIN BEFORE live NCP interaction each cycle — never unbounded growth; pendingCallbacks clears on `resetSession()`, ruled beat 58). **Dedup (measured contract):** consecutive-TSN + payload-equality, scoped per (device, endpoint, cluster), CLEARED on device-announce/rejoin (measured: EVERY SNZB event arrives ×2 within 8–21 ms; TSN is 8-bit wrapping AND resets on power-cycle — equality on (TSN, payload) within the scope, not TSN alone). Handler dispatch → typed attribute values → `state_reported` publishes through the injected `EventPublisher` (eventTime inheritance rules per event-model MODULE_CONTEXT — never `Instant.now()`).

**Cluster handlers:** OnOff (0x0006 → `on_off`), LevelControl (0x0008 → `brightness`), **ColorControl-CT with mireds→Kelvin AT INGESTION** (`K = 1,000,000 / mireds`; RAW MIREDS RETAINED per Doc 02 §3.7 auditability — AMD-96 Kelvin-canonical), OccupancySensing (0x0406 bit 0 → `occupancy.occupied` — the hero-trigger cluster), PowerConfiguration (raw/2 battery percent), **IAS Zone tolerate-not-require** (§3.12: CIE-write→enroll-response flow ATTEMPTED, enrollment recorded as a device fact, NEVER an interview gate; `ZoneStatusChangeNotification` ingested regardless of enrollment state — the SNZB-03P is dual-path with occupancy active, measured).

**Reporting configurator (Doc 08 §3.7 + charter):** bind-then-configure-then-**VERIFY**: ConfigureReporting per §3.7 defaults (including the AMD-96 OccupancySensing row) followed by ReadReportingConfiguration read-back; **the ACK-lies case is first-class** — configure accepted but read-back differs ⇒ posture DOWNGRADED honestly. Per-capability posture classification (VERIFIED_REPORTS/READBACK_ONLY/NONE × ON_CHANGE/PERIODIC/SLEEPY/NONE) recorded as DEVICE FACTS feeding AMD-97 `confirmability`. **Re-apply + re-verify on every rejoin/device-announce** (Hue power-loss rejoin + factory-reset config-wipe are the measured failure modes).

**Adoption slice (Doc 02 §3.12, identity-UNGATED per DP-B):** detection (`device_discovered` with protocol identity + proposed capabilities/entities) → proposal (DeviceIdentifiers dedup: IEEE match ⇒ re-link to existing device_id, `availability_changed`, NO new adoption event) → adoption (`device_adopted`, entity/capability registration through the existing integration-api surfaces).
