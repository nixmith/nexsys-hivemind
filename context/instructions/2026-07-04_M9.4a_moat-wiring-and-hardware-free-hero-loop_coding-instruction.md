### §1.2 Supersession expiry (F-2 / DP-c) — `StandardPendingCommandLedger`

### §1.3 The parameterized-expectation derivation (F-3 / DP-b) — the bridge
**Ctor/assembly change:** `StandardPendingCommandLedger` gains a `Function<String, Map<String, Object>> parameterDecoder` (nullable-forbidden; assembly threads it; lifecycle passes `persistenceFactory.commandParameterDecoder()` — P20 — at the existing assembly call site, HomeSynapseCore.java:613-621). Rule, exact:
1. Applicability: `instance.confirmation().mode()` ∈ {TOLERANCE, EXACT_MATCH} AND `instance.confirmation().authoritativeAttributes()` non-empty AND `definition != null` AND `definition.parameters()` has EXACTLY ONE required parameter of type INT/FLOAT (TOLERANCE) or BOOLEAN (EXACT_MATCH). 2. Decode `issued.parameters()` via the injected decoder; extract the single required parameter's value by its `parameterName`. Missing key / non-numeric where numeric expected / decoder throw → `Optional.empty()` (+ one DEBUG log naming entity, command, and why — Register C).
