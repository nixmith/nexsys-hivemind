## §2 — Brightness: the derivation-side root fix (SD-2)

### §2.1 Schema root-correction (device-model)
`StandardCapabilities.brightness()`: attribute schema → `intAttr("brightness", 0, 254, null, …)` (P47's 0,100 → 0,254). The `set_brightness` ParameterSchema stays `("level", INT, 0, 100, true, 0, null)` — percent is the user-facing capability domain (LevelControlHandler javadoc: *"the capability domain is percent; the 0–254 wire level exists only at this boundary"*). ConfirmationPolicy stays `(TOLERANCE, ["brightness"], 2, 5000L)` — the tolerance **2 is in LEVEL units** (Doc 08 §392: *"±2 of 128 … account for rounding differences between the 0–254 ZCL range"*), which post-fix is the attribute domain. Sweep device-model tests pinning the old bounds (P2 survey below).

### §2.2 The GENERIC param→attribute rescale (automation — `deriveOutcome`)
In the TOLERANCE branch (and structurally available to EXACT_MATCH numerics, though none exists today), after decoding the parameter value: look up the target attribute's `AttributeSchema` via `instance.attributes().get(attributeKey)`. **When BOTH the sourced `ParameterSchema` and the attribute schema carry non-null numeric `minimum` and `maximum` AND the bounds differ**, rescale:
```
target = attrMin + (decoded − paramMin) × (attrMax − attrMin) / (paramMax − paramMin)   // double math, Math.round to nearest
```
Otherwise identity (today's behavior — `set_color_temperature`'s equal 2000–6500 bounds take this leg unchanged; the M9.4a supersession/Kelvin pins stay green by construction). Tolerance is NOT rescaled — `policy.defaultTolerance` is attribute-domain by contract (§392). Schema-driven, zero ZCL knowledge, protocol-agnostic — exactly the ruled shape. **Worked example (format #5):** `set_brightness(50)` → param [0,100], attr [0,254] → target = round(50×254/100) = **127**; expectation `WithinTolerance(127, 2)`; the wire frame carries level 127 (LevelControlHandler: `Math.clamp(Math.round(50 × 254 / 100.0f), 0, 254)` = 127); the device reports 127 (or 128 — Hue rounding) → |reported−127| ≤ 2 → honest CONFIRMED. Before this fix the expectation was `WithinTolerance(50, 2)` vs a 127-level report — the grep-confirmed false-fail class, now dead.
**Alignment test (mandatory):** for every percent 0–100, assert |bridge target − LevelControlHandler wire level| ≤ 1 (both formulas round the same linear map; the ±2 tolerance absorbs the residual with margin).

### §2.3 Query-time percent derivation (state-store)
