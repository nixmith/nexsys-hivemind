# M9.4-PJ — Permit-Join Config Wiring (bench-blocker micro-WU)

**Why now (one paragraph):** the M9.4 bench acceptance requires joining two devices to the freshly formed network, and grounding confirmed there is NO production caller of `permitJoin` anywhere in main source — `CoordinatorProtocol.permitJoin(int)` (`CoordinatorProtocol.java:55`) is implemented (`EzspCoordinatorProtocol.java:600-608`, `FRAME_PERMIT_JOINING = 0x0022` :103) but reachable only from unit tests; `ZigbeeIntegrationAdapter.isPermitJoinActive()` is a stub (`:256-257`, comment: *"the permit-join surface binds with the REST path (post-M9.4a)"*); the config schema already documents `permit_join_duration` (1–254, `zigbee-config-schema.json`) but nothing reads it. This WU wires that key as the headless/bench operator path. The REST permit-join surface remains the future UI mechanism — this does not preempt it.

**North star:** never-false-ALIVE extends to the join window — `isPermitJoinActive()` must reflect the real window (clock-based), never claim open when closed. **Conservative default is LAW: key ABSENT ⇒ the window NEVER opens.** The schema's `default: 120` is documentation-side; the adapter must not auto-open without explicit operator config.

## Scope (exactly this, nothing else)

**§1 — Read the key.** In `ZigbeeIntegrationAdapter`'s PRODUCTION path only: after `resumeOrForm()` + `protocol.awaitNetworkUp()` succeed (`run()`, `:195-199`) and BEFORE the watchdog-armed cycle loop starts, read `context.configAccess().getInt("permit_join_duration")` (accessor exists: `ScopedConfigurationAccess.getInt`, returns `Optional<Integer>`; the section is already `integrations.zigbee`-scoped). Constant `PERMIT_JOIN_DURATION_KEY = "permit_join_duration"` beside `SERIAL_PORT_KEY` (`:59`).

**§2 — Open the window.** Key present: clamp to [1, 254] (defensive — schema validates upstream; on out-of-range log ONE WARN `zigbee.permit_join_clamped: configured={} clamped={}` and proceed with the clamp), call `protocol.permitJoin(n)` ONCE, record `permitJoinDeadline = clock.instant().plusSeconds(n)`, log INFO exactly `zigbee.permit_join_opened: duration={}s`. Key absent: no frame, no log, deadline stays null. NEVER in `initialize()` (INV-RF-03, `:44-45`); NEVER in the M9.4a driven/test mode (`runCycleOnce()` untouched).

**§3 — Honest window state.** `isPermitJoinActive()` (`:256-257`): replace the stub — `true` iff `permitJoinDeadline != null && clock.instant().isBefore(permitJoinDeadline)`. Keep the method's existing signature (it implements the public `ZigbeeAdapter` interface — zero public-surface change).

**§4 — Tests (before implementation, per section).** Home: `integration/integration-zigbee/src/test` (default-gated — runs in `check`; #13 grounding: the suite's existing `ZigbeeProductionTransportTest` substrate drives `run()` production mode already — extend it or sibling it):
1. Key present (e.g. 200) ⇒ exactly ONE `0x0022` frame with payload byte `[200]` sent AFTER network-up, none before.
2. Key absent ⇒ ZERO `0x0022` frames across the whole production boot.
3. Out-of-range (e.g. 999 via the raw map) ⇒ clamped frame `[254]` + the WARN.
4. `isPermitJoinActive()`: false before open; true inside the window; false after `Clock` advances past the deadline (fixed/mutable test clock — **§4c: no `Instant.now()`/`System.*time` anywhere in test code; inject `Clock`**).
5. Driven mode (`runCycleOnce()` path): zero `0x0022` frames — the M9.4a hero substrate is untouched.
