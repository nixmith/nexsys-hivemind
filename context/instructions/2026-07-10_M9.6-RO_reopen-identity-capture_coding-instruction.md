## What This Implements

The watchdog's reopen path searches enumerated ports for a candidate matching the identity CAPTURED at first open. With `integrations.zigbee.serial_port` pinned to an alias (`/dev/zigbee` — this bench's config, and the future udev best practice), `resolvePort()`'s synthesis arm (`ZigbeeIntegrationAdapter:405`) produces `PortCandidate(alias, null, -1, -1, null)` and `bindTransport()` captures `PortIdentity(0, 0, alias, EZSP)` (`:443`) — a value no enumerated candidate can EVER satisfy (Tier 1 compares the alias string against real `/dev/ttyUSB*`/by-id paths; Tier 2 compares 0/0 against real 0x10C4/0xEA60 or unknown −1/−1). Reopen has been structurally dead since boot on every alias-pinned config; field-confirmed on the soak boot (`production_session_started: port=/dev/zigbee`). This WU makes the CAPTURE truthful (resolve the alias to the real enumerated candidate at bind time) and gives the residual pinned-only identity an honest, NARROW reopen semantic (canonicalized-path-only matching). The strict-match safety intent survives untouched: reopen must never open the WRONG device.

## Settled Decisions (NOT open questions)

- **DP-1 (fix the capture — the root fix):** in `bindTransport()`'s `vendorId() < 0` arm: canonicalize the configured path (best-effort `Path.toRealPath()`; on any IOException fall back to the raw string) → re-scan `portEnumerator.enumerate()` for the candidate whose `systemPath()` equals the canonicalized path → capture `PortLocator.identityFor(thatCandidate, ...)` — a REAL identity (by-id stableId + real VID/PID). Only if still unfindable, fall back to the pinned-path identity (DP-3). The happy `vendorId() >= 0` arm is UNTOUCHED.
- **DP-2 (the pinned-only sentinel rule):** `PortLocator.reopenTarget` gains a leading arm: when `identity.isPinnedOnly()`, match by CANONICALIZED-PATH EQUALITY ONLY (identity.stableId() vs each candidate's systemPath/byIdPath, both sides canonicalized) — **Tier 2 is SKIPPED for pinned-only identities, and there is NO class-constant fallback**: with two coordinator-class sticks attached, an operator pin must never reopen the neighbor. Healthy identities flow through the existing two tiers BYTE-UNCHANGED.
- **DP-3 (PortIdentity honesty):** add package-private `boolean isPinnedOnly()` ⇒ `vendorId == 0 && productId == 0` — documented: USB VID 0x0000 is reserved/never a real device, so 0/0 is the sanctioned pinned-only sentinel (the ctor's negative-rejection stays; no relaxation). Javadoc states the semantic explicitly so 0/0 stops masquerading as a real USB id.
- **DP-4 (the canonicalizer seam):** `PortLocator` gains an injectable `UnaryOperator<String>` canonicalizer (ctor overload; production binding = the toRealPath-with-fallback function, defined once and shared with DP-1's bind-time use; tests inject pure maps — the class stays I/O-free under test, honoring its "stateless over an injected enumerator" contract).
- **DP-5 (observability, anti-vacuous):** ONE new INFO at bind time: `zigbee.port_identity_captured: stableId={} vendorId={} productId={} pinnedOnly={}` — the line that would have made the field diagnosis a one-glance read, and the both-topologies verification's glance-point. No other token changes; `zigbee.reopen_no_target` / `zigbee.reopened` / `production_session_started` BYTE-FROZEN.
- **DP-6 (REJECTED, recorded):** reopen falling back to full re-resolution (`locate()`/class constants) — REJECTED: it could open a foreign same-class stick and push the §2 NCP config batch into it on every backoff tick, with only the network-key guard between us and a neighbor's network. The strictness is the design.

## What to Watch Out For

- **Tests must inject `Clock`.** No `Clock.systemUTC()`/`Instant.now()`/`System.nanoTime()`/`System.currentTimeMillis()` in this module's test code — `Clock.fixed(...)` injected. (The locator family is already Clock-disciplined; keep it.)
- `toRealPath` is I/O — it belongs ONLY in the production canonicalizer binding, never in `PortLocator`'s core logic (the injectable seam is the point). bindTransport already does I/O; that arm may canonicalize inline via the shared function.
- The descriptor field stays LOGGING-ONLY (its documented ban from matching logic is a pin — do not be tempted).
- `attemptReopen()`'s post-match sequence (open → resetSession → startSession → resumeStored) is UNTOUCHED — this WU fixes finding the port, not what happens after.
- ReentrantLock/virtual-thread discipline unchanged; no new records expected (if one appears, the component/factory collision STOP-check applies).
