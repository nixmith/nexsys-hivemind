## What to Watch Out For

- **`state_confirmed` is never a log line** — anyone porting §51 legs by grepping logs for it writes a vacuously-red scenario. Verdicts ride the command lifecycle read.
- **Token freeze cuts both ways:** the scenarios bind frozen tokens (G-B1-3 verbatim); from this WU on, any core WU moving a token acquires a scenario-sweep obligation (charter §5) — your README states this so the obligation is discoverable.
- **The API token rotates per launch** — a restart stimulus mid-scenario invalidates the token you cached at scenario start; re-read after every `bench:` verb.
- **Organic traffic:** the SNZB-03P fires whenever it fires — at-least semantics everywhere except `exactly:` scoped to the run-window (DP-7).
- **journalctl permissions** on the Pi may deny the bench user; bundles degrade honestly (DP-6), never fail a scenario on evidence-collection trouble — but SAY SO in the manifest (absence-of-evidence recorded, the honesty doctrine applied to the instrument).
- **Disk:** bundles accumulate under `~/hs-bench/bundles/`; note the retention policy pointer in the README. Never write bundles into the repo tree.
- **No joins, no permit windows:** nothing in B1 touches pairing — the permit key stays OUT (Phase-7 anti-actions carry past the exit act until Nick's Wave-2 session).
- **Polling discipline:** poll-with-deadline per evidence line (the format's per-line `within:`); no global sleeps; no retry-until-green anywhere (scenario flake = a defect — charter §5).
