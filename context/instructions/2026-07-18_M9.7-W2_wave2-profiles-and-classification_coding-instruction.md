### §2 — The ingestion handlers

#### `TemperatureMeasurementHandler` (class, package-private, extends `ZigbeeClusterHandler`)
`CLUSTER_ID = 0x0402`; `ATTRIBUTE_MEASURED_VALUE = 0x0000`; invalid sentinel `0x8000`. Normalizes MeasuredValue (int16 signed, 0.01 °C) to the temperatureMeasurement capability's canonical attribute key and schema value type. Invalid/absent ⇒ empty list. Thread-safe: stateless over the device+clock binding (the sibling idiom).

#### `RelativeHumidityHandler` (class, package-private, extends `ZigbeeClusterHandler`)
`CLUSTER_ID = 0x0405`; `ATTRIBUTE_MEASURED_VALUE = 0x0000`; invalid sentinel `0xFFFF`. uint16, 0.01 %RH. Same contract shape.

**Behavioral contracts (both):** emit at most one NormalizedAttribute per report; raw wire value retained in the NormalizedAttribute raw slot (the existing idiom); no rounding beyond the schema's value type; sign-extension correct for negative temperatures (int16 — e.g. wire 0xF830 = −20.00 °C; embed this vector in a test).

#### `ClusterHandlers.forDevice` — 6 → 8 entries (add both; javadoc's handler list updated).

## What to Watch Out For
- **The QuantityValue ctor normalizes units** (`new QuantityValue(126,"K")` → −147.15 °C — the M9.3 gotcha). Match the capability schema's value type exactly; if the schema wants a plain numeric, use it — never wrap in a unit type the schema doesn't declare.
