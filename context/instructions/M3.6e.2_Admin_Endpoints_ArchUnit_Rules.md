## What This Implements

M3.6e.2 is the final sub-work-unit of the M3.6 composition-root milestone. It delivers three categories of work: (1) entity query REST endpoints exposing `StateQueryService` over HTTP, (2) admin/operational endpoints for DLQ inspection and projection status, and (3) ArchUnit rules enforcing the composition-root pattern. After M3.6e.2, the composition root is complete — HomeSynapse Core is a fully wired, externally queryable, operationally observable system.

M3.6e.1 (the prior sub-WU) delivered `MaterializedStateQueryService`, `ReadinessFilter`, `RestFilters` (DEC-M3-16 gateway), Javalin HTTP server bootstrap on port 7070, and `DeploymentProfile` thread pool sizing. M3.6e.2 builds on that foundation by adding the HTTP route handlers that call `StateQueryService` and the admin routes that inspect DLQ and projection state.

## Settled Decisions (from deliberation)

These decisions are NOT open questions. Implement exactly as stated.

- **SD-1 (ReadinessFilter scope):** `before("/api/*")` only. Admin/operational endpoints at `/internal/*` are NOT gated — they must respond even during REPLAY.
- **SD-2 (DEC-M3-16 gateway pattern):** All new public endpoint registration methods in `RestFilters` use `Object`-typed parameters to erase Javalin types from the public API surface. New handlers are package-private.
- **SD-3 (Port 7070 hardcoded):** Per PLAN-M3 §10. No configurability in M3.
- **SD-4 (No Jackson customization):** Endpoint handlers use Javalin's built-in JSON serialization (`ctx.json(...)`). Custom Jackson ObjectMapper configuration is Phase 3+ scope.
- **SD-5 (Admin endpoints on /internal/):** DLQ and projection status endpoints live under `/internal/` paths, NOT under `/api/v1/`. They are operational tooling, not public API.

## Technical Specification

### Entity Query Endpoints (per PLAN-M3 §10.6)

#### `ListEntitiesEndpoint` (package-private final class)
**Package:** `com.homesynapse.api.rest`
**Responsibility:** Handle GET /api/v1/entities — paginated entity listing
**Implements:** `io.javalin.http.Handler`

**Route:** `GET /api/v1/entities`
**Query params:**
- `limit` — int, default 50, max 100. Clamp silently (do not error).
- `sort` — `ASC` | `DESC`, default `ASC`. Case-insensitive.

**Response (200):**
```json
{
  "data": [ { "entityId": "...", "subjectType": "...", "availability": "...", "stale": false } ],
  "meta": { "viewPosition": 12345, "timestamp": "2026-05-22T..." }
}
```

**Headers:**
- `X-HomeSynapse-View-Position: {viewPosition}`

**Behavioral contracts:**
- Delegates to `StateQueryService.getAllStates()` (returns `Map<EntityId, EntityState>`)
- Sorts by EntityId string representation (ASC/DESC)
- Returns empty `data` array when no entities exist (NOT 404)
- `viewPosition` sourced from the StateQueryService's view-position supplier

#### `GetEntityEndpoint` (package-private final class)
**Package:** `com.homesynapse.api.rest`
**Responsibility:** Handle GET /api/v1/entities/{entityId} — single entity lookup
**Implements:** `io.javalin.http.Handler`

**Route:** `GET /api/v1/entities/{entityId}`
**Path param:** `entityId` — ULID string (Crockford Base32)

**Response (200):**
```json
{
  "data": { "entityId": "...", "subjectType": "...", "availability": "...", "attributes": {...}, "stale": false },
  "meta": { "viewPosition": 12345, "timestamp": "2026-05-22T..." }
}
```

**Response (404):** RFC 9457 problem detail with type from `ProblemType` enum.

**Behavioral contracts:**
- Delegates to `StateQueryService.getState(EntityId)` (returns `Optional<EntityState>`)
- Parse `entityId` path param via `EntityId.of(Ulid.fromString(param))` — if parse fails, return 400
- `stale` field is recomputed at read time (MaterializedStateQueryService handles this)

#### `GetEntityStateEndpoint` (package-private final class)
**Package:** `com.homesynapse.api.rest`
**Responsibility:** Handle GET /api/v1/entities/{entityId}/state — detailed entity state
**Implements:** `io.javalin.http.Handler`

**Route:** `GET /api/v1/entities/{entityId}/state`
**Path param:** `entityId` — ULID string

**Response:** Same shape as GetEntityEndpoint but returns the full `EntityState` record including all attributes.

**Behavioral contracts:**
- Same delegation as GetEntityEndpoint — both call `StateQueryService.getState(EntityId)`
- The distinction is cosmetic for MVP (both return EntityState). Future versions may return different projections.

### Admin/Operational Endpoints

#### `DlqStatusEndpoint` (package-private final class)
**Package:** `com.homesynapse.api.rest`
**Responsibility:** Handle GET /internal/dlq — dead letter queue status
**Implements:** `io.javalin.http.Handler`

**Route:** `GET /internal/dlq`
**NOT gated by ReadinessFilter** (operational endpoint, must respond during REPLAY).

**Response (200):**
```json
{
  "subscribers": [
    {
      "subscriberId": "state-projection",
      "parkedCount": 0,
      "oldestParkedAt": null
    }
  ]
}
```

**Behavioral contracts:**
- Reads DLQ state from the event bus's subscriber snapshots (`EventBus.subscribers()` → `List<SubscriberSnapshot>`)
- Each snapshot has a `dlqSize()` accessor (or equivalent)
- This is a read-only status view — no mutations

#### `ProjectionStatusEndpoint` (package-private final class)
**Package:** `com.homesynapse.api.rest`
**Responsibility:** Handle GET /internal/projection — state projection status
**Implements:** `io.javalin.http.Handler`

**Route:** `GET /internal/projection`
**NOT gated by ReadinessFilter.**

**Response (200):**
```json
{
  "mode": "LIVE",
  "viewPosition": 12345,
  "entityCount": 42,
  "ready": true
}
```

**Behavioral contracts:**
- `mode` from `ReadinessSource.mode()` → `SubscriberMode.name()`
- `viewPosition` from the view-position supplier
- `entityCount` from `StateQueryService.getAllStates().size()`
- `ready` from `readinessSource.mode() == LIVE`

## Locked Decisions That Apply

- **LTD-04 (ULID):** Entity IDs in URL paths are Crockford Base32 strings. Parse via `Ulid.fromString()` → `EntityId.of(ulid)`. Return 400 on parse failure.
- **LTD-08 (Jackson JSON):** Javalin's `ctx.json()` uses Jackson. No custom ObjectMapper configuration in M3.6e.2.
- **LTD-11 (No synchronized):** All handler classes must be stateless or use ReentrantLock. Endpoint handlers should be stateless (all state accessed via injected services).
- **DEC-M3-16 (Gateway pattern):** All new endpoint registration flows through `RestFilters` static methods. Handler classes are package-private.

## Invariants That Must Hold

- **INV-ES-02 (Derived state):** Entity state returned by endpoints is computed from the event log via StateProjection → MaterializedStateQueryService. No independent state storage.
- **INV-PR-01 (Constrained hardware):** Endpoint handlers must not buffer large result sets in memory. The `getAllStates()` map is bounded by entity count (MVP target: <500 entities).

## What to Watch Out For

1. **`-Xlint:exports` on new RestFilters methods.** Any new public method on `RestFilters` that takes a parameter type from a non-transitive `requires` module will fail `-Xlint:exports`. Pattern: use `Object` for types from `io.javalin` and `com.homesynapse.event.bus` (both are non-transitive in rest-api's module-info). Cast internally.

2. **Gradle/JPMS scope alignment.** If you add new `requires transitive` edges to rest-api's module-info, the corresponding build.gradle.kts dependency must be `api(...)`, not `implementation(...)`. M3.6e.1's second fix round was exactly this issue (OR-M3-16).

3. **ReadinessFilter does NOT gate /internal/* routes.** The `before("/api/*")` path filter means `/internal/dlq` and `/internal/projection` are always reachable. Verify this by testing DLQ/projection endpoints while ReadinessSource.mode() returns REPLAY — they should return 200, not 503.

4. **EntityId parsing from URL path.** `Ulid.fromString()` throws `IllegalArgumentException` on malformed input. Catch this and return 400 with a descriptive RFC 9457 body. Do NOT let it propagate as 500.

5. **`Map.copyOf` and null attribute values.** `EntityState.attributes()` may contain null values per the contract. If you serialize the attributes map via `ctx.json()`, Jackson handles nulls correctly (writes `null` values). Do NOT call `Map.copyOf()` on the attributes — it throws on null values.

6. **Thread safety of endpoint handlers.** Javalin handlers are called from Jetty's thread pool. The endpoint handler classes must be stateless — all state accessed via injected `StateQueryService`, `ReadinessSource`, etc. These are already thread-safe (StateQueryService delegates to ConcurrentHashMap-backed StateStore; ReadinessSource reads volatile SubscriberMode).

7. **ArchUnit rule package patterns.** ArchUnit's `resideInAPackage("com.homesynapse.api.rest..")` uses `..` for "any subpackage". Verify this matches the actual package structure. rest-api has subpackages (`com.homesynapse.api.rest.command`, etc.) — the `..` wildcard must capture all of them.

8. **NO_DIRECT_TIME_ACCESS compliance.** Endpoint handlers that need timestamps (for `meta.timestamp`) must use the injected `Clock`, not `Instant.now()`. Pass `Clock` through `RestFilters.installEntityQueryEndpoints(...)`.
