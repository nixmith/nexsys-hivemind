# M4.0b-3 Design Track — Typed Change Detection (real AMD-51 / AMD-52)

**Goal of M4.0b-3:** replace the M4.0b-1 **string** change-detect derivation rule with a **typed** one that compares `AttributeValue` variants by meaning, and (if decided) carry typed values in `StateChangedEvent`. This bumps `projectionVersion` **2→3**, riding AMD-50's reconciliation-backfill path unchanged.

**This is a *design* track, not a coding track yet.** The coding instruction cannot be written until the design problems below are deliberated and Nick makes the strategic calls. The point of this map is to make the path from "here" to "M4.0b-3 coding instruction" fully legible so the next Cowork conversation can execute it without re-discovery.

## 3. The riskiest design problem to scope early: AMD-52 serializer/replay blast radius

NQ-10-4 is the one that can balloon. Before the next Cowork session commits to a typed `StateChangedEvent`, it should map:

- **Serializer impact** — `CheckpointSerializer` currently flattens to string (M4.B3 noted it as an `instanceof`-with-fallback site, not an exhaustive switch). A typed payload needs a Jackson (de)serialization strategy for the 8-variant sealed hierarchy that is **deterministic across `projectionVersion`** and **upcaster-compatible** for old persisted events.
- **Event-store schema** — does the typed payload change the on-disk event row shape, or is it contained inside the existing payload JSON column? (Almost certainly the latter — confirm.)
- **Replay determinism** — replaying an old `state_changed` written with a string payload, under the new typed rule, must produce a deterministic typed value. This is the upcaster's load-bearing job and the precise interaction AMD-50's backfill + the upcaster must jointly satisfy.
- **Staging option** — AMD-51 (comparator) is independent of AMD-52 (typed event). If the serializer work is large, ship AMD-51 first (typed compare, string event) then AMD-52 (typed event) as a separate `projectionVersion` bump. Decide staging in the design beat.

This section is the single most important thing to resolve before writing the M4.0b-3 coding instruction.
