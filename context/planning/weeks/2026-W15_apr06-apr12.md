## The Arch-Debt Incident (Apr 11)

M2.2 (`696ac37`) introduced `Instant.now()` in `MigrationRunner`. M2.4 (`4b20786`) introduced `System.nanoTime()` in `JacksonWarmup`. Both live in `com.homesynapse.persistence`, which is not on the `NO_DIRECT_TIME_ACCESS` whitelist. Neither milestone's coder session ran `./gradlew check` — the project's policy is to defer full build verification to Nick's sandbox-external environment because the coder's sandbox cannot reliably run Gradle. The coder's unit tests passed in isolation because the ArchUnit rule runs in the app module's test task, not in persistence's.

The violations surfaced when the M2.5 test run caught them, requiring commit `d6a6065` before M2.5 could land. Root cause: **deferred build gates were not being tracked as open risks by the PM.** Fix: the Work Unit Completion Protocol now requires every deferred build gate to be logged on `pm-handoff.md` under "Open Risks" until resolved, and the freshness preflight will fail the session if a deferred gate is stale.
