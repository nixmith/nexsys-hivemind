**This protocol has been generalized and renamed.** The authoritative file is now `work-unit-completion-protocol.md` (WUCP), which covers both Phase 2 "blocks" and Phase 3 "milestones" uniformly.

**Go here:** [work-unit-completion-protocol.md](./work-unit-completion-protocol.md)

**Why the rename (2026-04-11):** Phase 3 uses "Milestone" vocabulary instead of "Block," and the BCP's Phase-2-specific language was causing ambiguity about whether the protocol applied to Phase 3 work. In practice, BCP Phase 2 (PM-side closeout) stopped running after Phase 3 began, causing ~3 weeks of governance drift. The generalized protocol fixes both issues — vocabulary-neutral wording and a new prime rule that makes work-unit completion a prerequisite for starting the next work unit.

See `context/audits/2026-04-11_m2.5-arch-debt-retrospective.md` for the full forensics.

The BCP vocabulary (Phase 1 / Phase 2) is preserved in WUCP — the phases do the same things, just with broader applicability.
