# ASIMTOTE — Company-Mark Companion Brief (for a combined-quote engagement)

## What this is

The client's brand architecture is a two-register model: **ASIMTOTE** as the parent/company mark (technical + corporate register; the engine ships as "Asimtote Core") and **TAMORO** as the consumer ecosystem mark (the main memo's subject). ASIMTOTE was locked as the company name 2026-06-13 and the client already owns its domain family. This brief packages the ASIMTOTE clearance question so counsel can quote it as a companion work-stream — one conflicts check, one engagement, both marks — on whatever timing counsel recommends. It is deliberately decoupled: nothing about TAMORO's Phase-1 timeline waits on it.

## The mark

- **ASIMTOTE** (standard characters) — coined; a phonetic respelling of "asymptote" (the silent p removed — the word as ears receive it). No dictionary meaning in any language checked; quiet positive morpheme ("Asim": Arabic/Hindi guardian/protector). One vetted foreign-reading flag set on file (Mandarin "si" homophone note; PT-BR "assim"), none judged disqualifying by the naming research (2026-06-13 assessment).
- Brand logic: the asymptote metaphor — approaches the ideal ever more closely, never claims arrival — is the product's documented honesty thesis (never-false-confirmed verdicts). Suggestive/fanciful, not descriptive.
- Styling: "Asimtote" (corporate) / `asimtote` (product/CLI), never lowercase-only. Word mark first.
- **Fallback if clearance fails: ASYMTOTE** (drop-p variant) — client already owns asymtote.com (screened registered on the client's own NS family, dns.google 2026-07-10); no exact ASYMTOTE mark surfaced (screen 2026-07-10; nearest formative: ASYMPTOTE GENETIC MEDICINES, SN 98915324, far-field).

## The named clearance questions (why this needs counsel, not more screening)

1. **Asimptote B.V.** (Netherlands) — simulation/thermal-power **software** firm, `asimptote.com`, one letter from the client's mark, same broad industry. The likeliest confusing-similarity vector, and it sits exactly where public-index screening cannot resolve: EU/BENELUX registers + phonetic equivalence. This is the named attorney question from the 2026-07-11 brief (§1 row 1, §3 risk 2).
2. **Phonetic-equivalents sweep** — asymptote/asimptote/assymtote and kin: requires TESS-grade phonetic search (US) + EUIPO/WIPO equivalents; public indexes cannot sweep it.
3. **The Cambridge tool as common-law use** — same string, software, same metaphor; assess whether a class-9/42 registration is constrained by it (client's read: different field of use — network-config diffing vs smart-home platform — and no consumer presence; test that read).
4. Classes: **9, 35, 42** (per the 2026-06-13 clearance-checklist scope) — counsel to align with the branded-house reality (the parent mark backstops the product line; "Asimtote Core" is the shipping engine string).

## Suggested engagement structure (counsel to price)

- Quote as a **companion/rider to the TAMORO Phase-1 engagement**: the marks share one client, one product record, one entity question, and one foreign-wave calendar — a second engagement later would re-buy all of that context.
